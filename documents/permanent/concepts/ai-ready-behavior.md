# AI-Ready Behavior

## 정의

AI-Ready Data(학습 데이터 품질 진단) 역량을 에이전트 스킬 문서 품질 진단으로 확장한 프레임워크. "데이터를 진단 가능한 상태로 만든다"는 명제를 "행동(스킬 문서)을 진단 가능한 상태로 만든다"로 전환한다.

## 맥락

페블러스(Pebblous)가 SkillOpt(arXiv:2605.23904) 분석을 통해 제시한 방향이다. 기존 DataClinic 5신호를 스킬 문서 5신호(SkillClinic)로 재맵핑한다:

| DataClinic (학습 데이터) | SkillClinic (스킬 문서) | 진단 질문 |
|---|---|---|
| 레이블 무결성 | 검증 무결성 | held-out gate 통과 이력이 깨끗한가? |
| 분포 균형 | 전이 가능성 | 다른 모델·하네스에서도 작동하는가? |
| 신선도 | 갱신 신선도 | 옵티마이저가 마지막으로 다듬은 게 언제인가? 죽은 스킬은 아닌가? |
| 결측 | 의미 누락 | 어떤 실패 모드를 아직 다루지 못하고 있는가? |
| 이상치 | 실행 이상 | rejected-edit buffer에 같은 종류 거절이 반복되는가? |

실무에서 조직이 묻게 되는 세 가지 질문: (1) 스킬 라이브러리 현황 — 살아있는 것·죽은 것·중복인 것의 수, (2) 검증 게이트 엄격성 — 옵티마이저가 수정한 스킬을 사람이 어떻게 검토하는가, (3) 모델 전환 시 스킬 전이 가능성 — 어떤 스킬이 전이되고 어떤 스킬이 깨지는가.

한국 AI 기본법(2026-01-22 시행)의 에이전트 책임 귀속 쟁점과 직결된다 — [[스킬-문서|에이전트 오판]] 시 어떤 스킬 문서의 어떤 줄에서 비롯되었는지를 사람이 읽을 수 있는 형태로 증명하는 능력이 [[에이전틱-ai]] 거버넌스의 새 표준이 된다는 시각이다. 행동 데이터베이스(에이전트 성공/실패 로그, 실행 궤적, 도구 호출 패턴)가 데이터 진단 자산과 자연스럽게 결합 가능하다는 점이 이 프레임워크의 확장 방향이다.
