# Java 추상클래스(Abstract Class)

## 정의

하위 클래스들의 공통점을 추상화해 계층 관계를 구축하고 중복 멤버를 통합·확장하는 Java 클래스 타입; `abstract` 키워드로 선언하며 단일 상속만 허용한다.

## 맥락

추상 메소드 외에 일반 필드·메서드·생성자를 자유롭게 가질 수 있다. 접근 제어자 제한이 없어 `protected`/`private` 멤버 선언이 가능하고, non-static·non-final 필드로 인스턴스별 상태 관리도 된다.

핵심 사용 목적은 **논리적 계층 구조 구축**이다. `extends` 키워드처럼 상위 클래스의 기능을 하위 클래스로 확장시키며, 클라이언트는 추상 클래스 타입으로 선언하고 업캐스팅으로 초기화해 다형성을 활용한다. 추상클래스는 클라이언트에서 자료형을 사용하기 전에 **미리 논리적인 클래스 상속 구조를 만들어 놓고 사용이 결정**되는 느낌이다.

**중복 멤버 통합**: 여러 클래스가 공통으로 가지는 필드·메소드를 추상클래스로 끌어올려 중복을 제거한다. 예: `NewlecExam`과 `YBMExam`이 모두 가진 `kor`·`eng`·`math` 필드와 `total()`·`avg()` 메소드를 `Exam` 추상클래스로 통합.

**명확한 계층 구조**: "삼각형·원→도형", "사자·고양이→동물"처럼 is-a 관계가 명확할 때 사용한다. SMS Sender 사례에서 각 통신사별 커넥션 구현(`establishConnectionWithYourTower()`)은 다르지만, 공통 방해금지 모드 체크 로직은 추상클래스 `SMSSender`에서 공유한다.

인터페이스와 조합 시 **Interface-Abstract-Concrete 패턴**으로 발전한다. 인터페이스가 다중 타입 묶음을 보장하고, 중간 추상클래스가 공통 필드·메소드(예: `opacity`·`color`)를 통합해 구체 클래스의 중복을 제거한다. 단, 이미 다른 클래스를 상속받아야 한다면 이 패턴을 활용할 수 없다.
