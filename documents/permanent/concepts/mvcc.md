# MVCC(다중 버전 동시성 제어)

## 정의

데이터베이스에서 락(Lock) 없이 데이터의 여러 버전을 관리하여 읽기와 쓰기가 서로를 차단하지 않고 동시에 진행될 수 있게 하는 동시성 제어 기법; 트랜잭션마다 시작 시점의 스냅샷을 독립적으로 조회하게 함으로써 직렬 가능성(Serializability)을 보장한다.

## 맥락

MVCC는 트랜잭션 시작 시 부여된 **타임스탬프**와 데이터의 버전별 타임스탬프를 비교하여 해당 시점에 맞는 버전을 제공하는 방식으로 동작한다.

업데이트 발생 시 기존 데이터는 **언두(Undo) 영역**에 백업되고, 원래 공간은 새 데이터로 교체된다. 커밋이 완료되면 **SCN(System Change Number)**이 1 증가한다. 동시에 읽기를 수행하는 다른 트랜잭션은 언두 영역에서 자신의 타임스탬프 이전 버전을 찾아 **CR카피(Consistent Read Copy)**로 제공받으므로 쓰기 작업과 충돌하지 않는다.

세 가지 핵심 구성 요소:
- **SCN**: 커밋마다 증가하는 시스템 전역 시계
- **언두 영역**: 변경 전 버전을 보관하는 저장소
- **CR카피**: SCN 기준으로 쿼리 시점에 맞는 과거 버전을 조립한 읽기 전용 복사본

트레이드오프로 I/O 오버헤드(언두 영역 디스크 접근 증가), 스토리지 오버헤드(버전 누적 보관), CPU 오버헤드(버전 선택 계산)가 발생한다. Oracle, PostgreSQL 등 현대 관계형 데이터베이스 대부분이 MVCC를 채택하고 있으며, 쿼리 성능 저하 진단 시 "언두 블록 I/O 과부하" 여부를 구체적으로 확인하는 관점이 실무에서 유용하다.
