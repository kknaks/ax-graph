# 구조 패턴(Structural Patterns)

## 정의

GoF 디자인 패턴의 구조 범주; 클래스·인터페이스·객체를 조합해 더 큰 구조를 만들며, 기존 코드를 최소한으로 변경하면서 기능을 추가할 수 있게 하는 설계 패턴 집합이다.

## 맥락

[[gof-디자인-패턴]]의 3범주 중 두 번째로, 강의에서 특히 강조되는 대표 패턴은 다음과 같다:

| 패턴 | 핵심 목적 |
|---|---|
| 어댑터(Adapter) | 호환되지 않는 인터페이스를 함께 사용할 수 있게 변환 |
| 브릿지(Bridge) | 구현과 추상화를 분리해 독립적으로 확장 |
| 데코레이터(Decorator) | 객체에 동적으로 새 기능을 추가 |
| 파사드(Facade) | 복잡한 서브시스템에 단순한 인터페이스 제공 |

구조 패턴의 핵심 가치는 기존 코드베이스를 최소 수정하면서 새로운 기능 요구사항에 대응할 수 있는 **개방-폐쇄 원칙(Open-Closed Principle)** 실천이다.
