# 생성 패턴(Creational Patterns)

## 정의

객체 생성과 관련된 GoF 디자인 패턴 범주; 특정 객체가 생성되거나 변경되어도 프로그램 구조에 최소 영향을 주는 유연성을 제공하는 5가지 패턴으로 구성된다.

## 맥락

[[gof-디자인-패턴]]의 3범주 중 첫 번째로, 다음 5가지 패턴을 포함한다:

| 패턴 | 핵심 목적 |
|---|---|
| 추상 팩토리(Abstract Factory) | 관련 객체군을 구체 클래스 지정 없이 생성 |
| 빌더(Builder) | 복잡한 객체를 단계별로 생성 |
| 팩토리 메소드(Factory Method) | 객체 생성을 하위 클래스에 위임 |
| 프로토타입(Prototype) | 기존 객체를 복사해 새 객체 생성 |
| [[싱글톤-패턴]](Singleton) | 클래스 인스턴스를 하나만 생성해 전역 참조 제공 |

외부 API 사용 시 이미 적용되어 있어 개발자가 자연스럽게 접하게 되는 패턴들이 포함된다. 스프링 부트와 가장 직결되는 패턴은 [[싱글톤-패턴]]으로, 스프링의 빈(Bean) 관리 방식이 싱글톤 원칙에 기반한다.
