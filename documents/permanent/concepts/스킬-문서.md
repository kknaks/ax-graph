# 스킬 문서(Skill Document)

## 정의

에이전트가 특정 작업을 수행하는 방법을 자연어로 기술한 이식 가능한 아티팩트. 절차(procedures), 도메인 휴리스틱, 도구 사용 정책, 출력 제약, 실패 모드를 담은 마크다운 파일(best_skill.md)이며, 모델·실행환경에 종속되지 않고 학습·검증·내보내기·재사용이 가능하다.

## 맥락

Microsoft Research의 SkillOpt(arXiv:2605.23904, 2026-05-22)가 스킬 문서를 학습 가능한 단일 최적화 대상으로 정식화했다. 기존 에이전트 최적화가 모델 가중치(연속·고차원·모델 종속·인간 불가독)를 조정 대상으로 삼았다면, 스킬 문서는 자연어 공간(이산·의미적·모델 독립·인간 가독)에 위치한다.

세 가지 핵심 특성:

1. **인간 가독성**: GB짜리 모델 파일 대신 수 KB의 마크다운. 변경 이력을 git으로 추적하고 사람의 검토 게이트를 삽입할 수 있어, "왜 이렇게 결정했는가"를 텍스트로 묻고 답할 수 있다 — AI 거버넌스·책임 귀속의 새 표준.
2. **모델 독립적 이식성**: GPT-5.4에서 학습한 스킬을 Qwen3.5-4B로, Codex 환경 스킬을 Claude Code 환경으로 전이 가능. SkillOpt 실험 기준 Cross-model +15.2%, Cross-harness +31.8%, Codex→Claude Code SpreadsheetBench 22.1→81.8(+59.7pt).
3. **누적 자본재 위상**: 모델 가중치가 매 세대마다 재작성해야 하는 소모재인 반면, 스킬 문서는 한 번 학습하면 다음 모델 세대로 이전 가능한 누적 자산이다.

[[루프-엔지니어링]]의 5요소 중 스킬(Skills) 계층이 곧 스킬 문서다 — 없으면 루프가 매 사이클마다 프로젝트 전체를 처음부터 재파생한다. SkillOpt는 이 계층을 epoch·mini-batch·학습률·검증 게이트 등 딥러닝 훈련 규율로 자동 최적화해 정적 인간 작성 자산에서 자기진화 자산으로 전환한다.

[[에이전틱-ai]] 시스템에서 스킬 문서는 frozen 에이전트와 frozen 모델 사이에 위치한 텍스트 한 장이다. 옵티마이저 모델(별도 LLM)이 타깃 모델의 실행 궤적을 분석해 add·delete·replace 편집을 제안하고, held-out validation gate를 통과할 때만 채택된다.

**스킬 파일의 실측 특성** (SkillOpt 6개 케이스 스터디 기준): 최종 배포 스킬 파일의 중앙값 길이는 **약 920토큰**이다. 검증 게이트가 대부분의 편집 제안을 기각하기 때문에 최종 파일에 채택되는 편집은 **1~4개**에 불과하다 — OfficeQA 태스크에서 +39.0점 향상이 단 1개의 채택 편집에서 비롯된 사례가 대표적이다. 학습된 규칙은 "숙련된 실무자의 조언"처럼 읽힌다.

**컴포넌트 제거(Ablation) 시사점**: 기각 편집 버퍼를 제거하면 검증한 3개 벤치마크 모두 점수가 하락한다. 메타 스킬 업데이트와 슬로우 업데이트를 함께 제거하면 SpreadsheetBench가 77.5 → 55.0으로 하락한다. 이는 기각 피드백과 에포크 단위 통합 모두가 수렴에 필수적임을 보여준다.
