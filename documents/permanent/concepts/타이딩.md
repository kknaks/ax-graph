# 타이딩(Tidying)

## 정의

켄트 벡(Kent Beck)이 정의한 의미 변경 없는 안전한 소규모 코드 정리 — 코드 뭉치 사이 빈 줄 추가, 코드 위치 이동 등 작은 단위의 구조 개선으로 코드베이스 품질을 점진적으로 높인다.

## 맥락

타이딩은 기능 변경 없이 코드의 가독성·일관성을 향상시키는 리팩토링의 안전한 하위 집합이다. 소규모이므로 리뷰 부담이 낮고 실수 위험이 적어 지속적으로 반복할 수 있다. 타이딩이 충분히 축적되면 팀이 더 대담한 구조 변경 결정을 내릴 수 있는 기반이 만들어진다.

코르카 문라이트 코드베이스 개선 사례에서 타이딩은 AI 워크플로우와 결합되었다: Claude Code 커맨드로 프롬프트를 등록하고, 셸 스크립트(tidy-target-commit)로 최근 커밋 중 타이딩 대상을 필터링하며, 병렬 서브 에이전트가 커밋 단위로 타이딩 제안을 생성한다. 단순 코드 전체가 아닌 최근 커밋을 분석 대상으로 설정한 것이 팀의 관심도와 임팩트를 높이는 설계 포인트였다.

코드베이스 품질 향상이 AX 전략으로 연결되는 이유: 정렬된 임포트, 일관된 코드 스타일 등 품질이 높아질수록 AI가 유사한 고품질 패턴을 참조할 확률이 증가하고 AI의 실수 확률이 감소한다.
