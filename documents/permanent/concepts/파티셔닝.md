# 파티셔닝(Partitioning)

## 정의

단일 DB 인스턴스 내부에서 하나의 거대한 테이블을 여러 파티션(저장 단위)으로 분리해 조회 성능과 관리 효율을 높이는 내부 최적화 전략. DB 엔진(옵티마이저)이 파티션 라우팅을 자동 처리하므로 개발자는 SQL 변경 없이 투명하게 사용할 수 있다.

## 맥락

데이터 폭증으로 단일 테이블의 IO 병목·동시성 충돌·운영 비용이 한계에 달할 때 [[샤딩]](물리적 다중 인스턴스 분산)에 앞서 먼저 검토하는 단일 서버 수준의 해결책이다.

파티셔닝 종류는 네 가지다:

1. **레인지(Range) 파티션** — 날짜·숫자 범위 기준 분할. 월별·연도별 로그 테이블에 적합(게임 회사 로그 테이블 실사례).
2. **리스트(List) 파티션** — 특정 값의 목록 기준 분할. 지역 코드(서울·부산 등) 분리에 적합.
3. **해시(Hash) 파티션** — 특정 컬럼에 해시 함수 적용 → 균등 분산. 특정 값 쏠림 방지 목적.
4. **컴퍼지트(Composite) 복합 전략** — 두 가지 이상 전략 조합(예: 범위 + 해시 혼합).

비즈니스 도메인 특성·데이터 규모·데이터 특성에 따라 적절한 전략을 선택한다.

실무 적합 사례:
- **로그 테이블**: 시간 기준 누적 대용량 테이블 → 날짜 컬럼 조회 성능 향상
- **대량 오래된 데이터 일괄 삭제**: 파티션 단위 DROP으로 효율적 정리
