# 콘텐츠 제작 자동화

> 한 줄 정의: 마케터가 홍보 주제를 입력하면 / 시스템이 SNS용 이미지·영상 콘텐츠를 AI로 생성해 / 바로 쓸 수 있는 비주얼 자산을 얻는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 원문은 AI 활용 대상으로 "SNS 홍보용 콘텐츠 (이미지 및 영상)" 제작을 첫 항목으로 든다. 지금은 SNS 홍보에 쓸 이미지·영상을 사람이 매번 손으로 기획·제작해야 해, 물량과 속도를 맞추기 어렵고 반복 작업에 시간이 든다.
- **필요 이유**: 홍보 이미지·영상 생성을 AI로 자동화해 사람이 기획·검수에만 집중하도록 만든다.

## 2. 기능 정의
- 홍보 주제·문구·타깃 채널을 입력받아 콘텐츠 생성 요청을 만든다.
- 입력을 바탕으로 SNS용 홍보 이미지를 AI로 생성한다.
- 입력을 바탕으로 SNS용 홍보 영상을 AI로 생성한다.
- 생성 결과를 사용자가 검토·수정 후 확정하도록 미리보기로 제시한다.
- 확정된 콘텐츠를 SNS 홍보에 쓸 자산으로 저장한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 홍보 주제·문구와 채널(예: 인스타)을 입력하고 이미지·영상 유형을 고른다 | 입력을 콘텐츠 생성 요청으로 만들어 AI로 이미지·영상을 생성하고 미리보기로 보여준다 | 확정된 SNS 홍보용 이미지·영상 자산 |
| 예외 | 주제 입력이 비었거나 생성이 실패한다 | 필수 입력을 안내하거나 실패 사유를 알리고 재생성 경로를 준다 | 오류 안내 + 재시도 요청 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "SNS 홍보용 콘텐츠 (이미지 및 영상)"
- **기대 출력**: 입력한 홍보 주제에 맞는 SNS 게시용 이미지 1건과 짧은 홍보 영상 1건.
- **입력**: "콘텐츠 제작"
- **기대 출력**: 주제·채널 입력만으로 초안 비주얼을 생성해 검토용 미리보기로 제시.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. SNS 홍보용 이미지를 AI로 생성할 수 있어야 한다.
2. SNS 홍보용 영상을 AI로 생성할 수 있어야 한다.
3. 홍보 주제·문구·채널을 입력으로 받아 생성 요청을 구성해야 한다.
4. 생성 결과를 확정 전에 미리보기로 제시하고 재생성을 허용해야 한다.
5. 확정된 콘텐츠를 재사용 가능한 자산으로 저장해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 홍보 주제·문구·타깃 채널·콘텐츠 유형.
- 출력/저장: 생성된 SNS 홍보용 이미지·영상 파일.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 주제·문구 입력만으로 SNS용 이미지가 1건 이상 생성된다.
- [ ] 주제·문구 입력만으로 SNS용 영상이 1건 이상 생성된다.
- [ ] 생성 결과가 확정 전 미리보기로 제시되고 재생성이 가능하다.
- [ ] 확정된 콘텐츠가 자산으로 저장되어 다시 불러올 수 있다.
