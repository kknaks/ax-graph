# CS 응대 자동화

> 한 줄 정의: 고객이 문의를 남기면 / 시스템이 간단·야간 문의에 자동 응대하고 문의 데이터를 분석·정리하여 / 담당자가 반복 응대 부담 없이 문의 현황을 파악한다.

## 1. 요구 배경
- **현재 불편(as-is)**: 반복되는 간단한 문의와 담당자가 없는 야간 시간대 문의를 사람이 일일이 응대해야 해 응대 공백과 업무 부하가 생긴다. 원문은 CS 항목으로 "간단한 CS 응대"와 "야간 CS 응대 등"을 명시한다.
- **필요 이유**: 자동 응대와 문의 데이터 정리를 통해 반복·야간 문의를 시스템이 처리하고 담당자는 판단이 필요한 문의에 집중하게 한다.

## 2. 기능 정의
- **분류한다**: 들어온 문의를 간단 문의/에스컬레이션 대상으로 구분한다.
- **응대한다**: 간단한 CS 문의에 자동으로 답변을 생성해 응대한다.
- **야간 처리한다**: 담당자 부재 시간대(야간) 문의를 자동 응대하거나 접수·보류 처리한다.
- **정리한다**: 접수된 문의 데이터를 수집해 유형·빈도 기준으로 분석·정리한다.
- **이관한다**: 자동 응대가 어려운 문의는 담당자에게 전달한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 고객이 간단한 문의를 남긴다 | 문의를 간단 문의로 분류하고 자동 답변을 생성한다 | 자동 응대 답변, 문의 기록 |
| 정상(야간) | 고객이 야간에 문의를 남긴다 | 야간 시간대임을 인식해 자동 응대 또는 접수 처리한다 | 야간 자동 응대/접수 확인 |
| 예외 | 고객이 판단이 필요한 복잡한 문의를 남긴다 | 자동 응대 대상이 아니라고 판정해 담당자에게 이관한다 | 담당자 이관 알림, 보류 안내 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "간단한 CS 응대"에 해당하는 반복 문의
  - **기대 출력**: 사전 정의된 답변 기준에 따라 자동 생성된 응대 답변.
- **입력**: "야간 CS 응대 등"에 해당하는 야간 시간대 문의
  - **기대 출력**: 야간 자동 응대 또는 접수 확인 후 익일 담당자 확인 대기 처리.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 들어온 문의를 간단 문의와 이관 대상으로 분류해야 한다.
2. 간단한 CS 문의에 자동 응대 답변을 생성해야 한다.
3. 야간 시간대 문의를 인식해 자동 응대 또는 접수 처리해야 한다.
4. 문의 데이터를 수집·분석해 유형·빈도 등으로 정리해야 한다.
5. 자동 응대가 불가한 문의는 담당자에게 이관해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 고객 문의 내용, 문의 시각. 저장: 문의 이력·분류 결과·응대 기록. 출력: 자동 응대 답변, 문의 분석 정리 자료.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 간단 문의로 분류된 문의에 자동 응대 답변이 생성된다.
- [ ] 야간 시간대 문의가 자동 응대 또는 접수 처리된다.
- [ ] 자동 응대 불가 문의가 담당자에게 이관된다.
- [ ] 수집된 문의 데이터가 유형·빈도 기준으로 정리되어 확인 가능하다.
