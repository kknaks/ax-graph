# 마케팅 순위 체크

> 한 줄 정의: 마케터가 대상 검색어를 등록하면 / 시스템이 네이버 지도·블로그·체험단 순위를 매일 자동으로 조회해 / 마케터가 리포트로 결과를 받는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 마케팅 순위 확인이 "매일 사람이 직접 검색해서 확인하던 순위 체크 작업"으로 이뤄진다. 네이버 지도(플레이스)·블로그 상위 노출·체험단 글 순위를 사람이 매일 하나씩 검색해 손으로 기록해야 해서 반복적이고 시간이 든다.
- **필요 이유**: 반복되는 순위 검색을 시스템이 대신 수행해 마케터의 루틴 수작업을 없앤다.

## 2. 기능 정의
- 마케터가 등록한 검색어/대상에 대해 네이버 지도(플레이스) 순위를 조회한다.
- 등록된 검색어에 대해 블로그 상위 노출 순위를 조회한다.
- 체험단 글의 검색 순위를 조회한다.
- 위 조회를 매일 정해진 시각에 자동으로 실행한다.
- 조회 결과를 순위 리포트 형태로 정리해 제공한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 순위를 추적할 검색어·대상(플레이스/블로그/체험단 글)을 등록한다 | 매일 지정 시각에 각 항목의 순위를 자동 조회하고 결과를 집계한다 | 항목별 오늘 순위가 담긴 일일 리포트 |
| 예외 | 등록한 검색어의 순위를 조회한다 | 대상 글이 순위권에 없거나 조회에 실패하면 해당 항목을 미노출/실패로 표시한다 | 미노출·조회실패 사유가 표시된 리포트 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "네이버 지도(플레이스) 순위", "블로그 상위 노출 순위", "체험단 글 순위" 항목을 추적 대상으로 등록.
  - **기대 출력**: 각 항목의 당일 순위가 정리된 리포트.
- **입력**: "위 항목들을 매일 자동으로 체크해서 리포트로 보여줬으면 합니다."
  - **기대 출력**: 사람이 검색하지 않아도 매일 자동 생성되어 전달되는 순위 리포트.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 순위를 추적할 검색어·대상을 등록·관리할 수 있어야 한다.
2. 네이버 지도(플레이스) 순위를 조회할 수 있어야 한다.
3. 블로그 상위 노출 순위를 조회할 수 있어야 한다.
4. 체험단 글 순위를 조회할 수 있어야 한다.
5. 위 조회를 매일 자동으로(사람 개입 없이) 실행해야 한다.
6. 조회 결과를 항목별로 정리한 리포트로 제공해야 한다.
7. 순위권에 없거나 조회에 실패한 항목은 미노출/실패로 구분해 표시해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 추적 대상 검색어·플레이스/블로그/체험단 글 식별 정보.
- 저장: 항목별 일자별 순위 기록.
- 출력: 항목별 당일 순위가 담긴 일일 리포트.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 등록된 검색어에 대해 네이버 지도(플레이스)·블로그·체험단 세 항목의 순위가 조회된다.
- [ ] 사람이 수동 실행하지 않아도 매일 지정 시각에 순위 조회가 자동 실행된다.
- [ ] 조회 결과가 항목별로 정리된 리포트로 산출된다.
- [ ] 순위권에 없거나 조회 실패한 항목이 미노출/실패로 구분 표시된다.
