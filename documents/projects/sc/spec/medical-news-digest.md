# 의료 뉴스 요약

> 한 줄 정의: 마케터가 매일 아침 접속하면 / 시스템이 자동 수집·요약한 의료 뉴스 매거진을 열어 / 최신 의료 이슈 핵심을 한눈에 얻는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 의료 관련 최신 소식을 파악하려면 사람이 직접 여러 매체를 돌며 기사를 찾아 읽어야 해 시간이 든다. 원문은 "매일 나오는 의료 관련 뉴스들을 자동으로 수집하고, 핵심만 짧게 요약해서 매거진처럼 보여주면 좋겠습니다"라고 요구한다.
- **필요 이유**: 수집과 요약을 시스템이 대신하면 마케터는 읽을 것만 골라 빠르게 최신 흐름을 파악할 수 있다.

## 2. 기능 정의
- **핵심 동작**: 매일 정해진 시각에 의료 관련 최신 기사를 자동으로 수집한다.
- **핵심 동작**: 수집한 각 기사의 핵심만 짧게 요약한다.
- **핵심 동작**: 요약된 기사들을 매거진 형태의 리포트로 묶어 보여준다.
- **핵심 동작**: 매일 새 리포트를 갱신해 최신 회차를 제공한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 아침에 접속해 오늘의 의료 뉴스 매거진을 연다 | 당일 수집·요약된 기사를 매거진 형식으로 조립한다 | 핵심 요약이 담긴 오늘자 뉴스 매거진 |
| 예외 | 수집할 신규 기사가 없는 날 접속한다 | 새 기사가 없음을 확인하고 갱신을 건너뛴다 | "오늘 새 기사 없음" 안내 또는 직전 회차 표시 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "매일 나오는 의료 관련 뉴스들을 자동으로 수집하고"
  - **기대 출력**: 당일 발행된 의료 관련 기사 목록이 자동으로 모여 있는 상태.
- **입력**: "핵심만 짧게 요약해서 매거진처럼 보여주면 좋겠습니다"
  - **기대 출력**: 각 기사의 요지가 짧게 정리되어 매거진 레이아웃으로 배열된 리포트.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 시스템은 매일 자동으로 의료 관련 최신 기사를 수집해야 한다.
2. 수집한 각 기사는 핵심만 짧게 요약해야 한다.
3. 요약 결과는 매거진 스타일의 리포트로 묶어 제공해야 한다.
4. 리포트는 매일 최신 회차로 갱신되어야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 외부 의료 관련 기사(제목·본문·출처).
- 저장: 수집 기사와 생성된 요약.
- 출력: 매거진 형식의 요약 리포트.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 사람이 별도로 실행하지 않아도 매일 기사 수집이 자동으로 이뤄진다.
- [ ] 각 기사에 원문보다 짧은 핵심 요약이 붙는다.
- [ ] 당일 요약이 매거진 형태의 리포트 한 화면으로 제공된다.
- [ ] 신규 기사가 없는 날에도 오류 없이 상태를 안내한다.
