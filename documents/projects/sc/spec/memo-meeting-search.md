# 메모·회의록 검색

> 한 줄 정의: 사용자가 자연어로 질의하면 / 자유롭게 쌓인 메모·회의록에서 원하는 내용을 검색하면 / 정확한 해당 기록을 답변으로 받는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 대화창 한곳에 메모와 회의록을 자유 형식으로 계속 쌓아두다 보니, 나중에 특정 기록을 찾을 때 어디에 적었는지 헷갈린다. 원문은 "아무 대화창에나 자유롭게 회의록이나 메모를 막 적어둬도, 나중에 원하는 내용만 쏙쏙 찾아주는 기능"을 원한다고 적고 있다.
- **필요 이유**: 자연어 질의로 흩어진 기록에서 정확한 항목만 골라 반환하면, 기록 위치를 기억하거나 수동으로 뒤질 필요가 없어진다.

## 2. 기능 정의
- 자유 형식으로 입력된 메모·회의록을 검색 가능한 형태로 저장한다.
- "7월 7일 SC 임원회의 내용 찾아줘"와 같은 자연어 질의를 해석해 날짜·대상·주제 등 검색 조건을 추출한다.
- 추출한 조건으로 저장된 기록을 검색해 질의에 부합하는 항목을 찾는다.
- 여러 기록이 섞여 있어도 헷갈리지 않고 질의에 맞는 정확한 기록을 반환한다.
- 질의에 해당하는 기록을 답변 형태로 사용자에게 제시한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | "7월 7일 SC 임원회의 내용 찾아줘"라고 질의한다 | 날짜(7월 7일)·대상(SC 임원회의) 조건을 추출해 저장된 기록을 검색한다 | 해당 임원회의 메모·회의록 내용을 반환한다 |
| 예외 | "지난주 A고객사 만난 내용 보여줘"라고 질의했으나 해당 기록이 없다 | 조건에 맞는 기록을 검색했으나 일치 항목을 찾지 못한다 | 해당 기록이 없음을 알리고 유사·인접 기록을 제시한다 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "7월 7일 SC 임원회의 내용 찾아줘"
  - **기대 출력**: 7월 7일 SC 임원회의로 기록된 메모·회의록 내용을 정확히 반환.
- **입력**: "지난주 A고객사 만난 내용 보여줘"
  - **기대 출력**: 지난주 A고객사 미팅으로 기록된 내용을 헷갈리지 않고 정확히 반환.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 아무 대화창에나 자유 형식으로 적어둔 메모·회의록도 검색 대상으로 저장해야 한다.
2. 자연어 질의에서 날짜·대상·주제 등 검색 조건을 해석해야 한다.
3. 여러 기록이 한곳에 쌓여 있어도 질의에 맞는 정확한 기록을 헷갈리지 않고 반환해야 한다.
4. 질의에 부합하는 기록이 없으면 없음을 알려야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 사용자의 자연어 검색 질의.
- 저장: 자유 형식으로 입력된 메모·회의록 기록(날짜·대상·내용 등).
- 출력: 질의에 부합하는 메모·회의록 검색 결과.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] "7월 7일 SC 임원회의 내용 찾아줘" 질의에 해당 임원회의 기록을 반환한다.
- [ ] 자유 형식으로 입력한 메모·회의록이 검색 대상으로 저장된다.
- [ ] 여러 기록이 섞인 상태에서 질의 대상과 다른 기록을 반환하지 않는다.
- [ ] 일치하는 기록이 없을 때 없음을 명시적으로 알린다.
