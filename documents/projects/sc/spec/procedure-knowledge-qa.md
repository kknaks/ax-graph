# 시술 지식 Q&A

> 한 줄 정의: 신입사원이 피부·성형 시술이 궁금하면 / 자연어로 물어보면 / 사전 입력된 지식에서 바로 답을 얻는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 신입사원이 시술·수술에 대해 궁금한 점이 생겨도 즉시 물어볼 창구가 없다. 원문은 "신입사원이 시술이나 수술에 대해 궁금한 점을 물어보면 바로 알려주는 기능"과 "신입사원이 묻는 말에 친절하게 답해주는 검색/대화창이 필요합니다"라고 요구한다.
- **필요 이유**: 피부과·성형외과 주요 시술의 특징·주의사항을 사전에 데이터로 넣어두고 자연어 질문에 답하면, 신입사원이 별도 문의 없이 스스로 지식을 확인할 수 있다.

## 2. 기능 정의
- **핵심 동작**: 피부과·성형외과 주요 시술의 특징·주의사항을 지식 데이터로 사전 입력받아 저장한다.
- 신입사원이 입력한 자연어 질문을 받아 관련 시술 지식을 검색한다.
- 검색된 지식을 근거로 질문에 대한 답변을 대화창으로 제시한다.
- 관련 지식이 없거나 모호하면 답을 지어내지 않고 확인 불가를 알린다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 신입사원이 대화창에 시술 관련 질문을 입력한다 | 사전 입력된 시술 지식에서 관련 항목을 검색해 답변을 생성한다 | 시술 특징·주의사항이 담긴 답변 |
| 예외 | 사전 입력 데이터에 없는 시술을 질문한다 | 관련 지식을 찾지 못함을 판정한다 | 답을 지어내지 않고 "해당 정보 없음"을 안내 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "신입사원이 시술이나 수술에 대해 궁금한 점을 물어보면" — 예: 특정 시술의 주의사항을 자연어로 질문.
- **기대 출력**: 사전 입력된 해당 시술의 특징·주의사항을 근거로 한 친절한 답변.
- **입력**: "피부과, 성형외과 주요 시술의 특징, 주의사항 등을 데이터로 넣어두고" 기반의 질문.
- **기대 출력**: 저장된 지식에서 검색된 해당 시술 설명.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 피부과·성형외과 주요 시술의 특징·주의사항을 지식 데이터로 사전 입력·저장할 수 있어야 한다.
2. 신입사원의 자연어 질문을 검색/대화창으로 받아야 한다.
3. 저장된 시술 지식에서 질문과 관련된 항목을 검색해야 한다.
4. 검색 결과를 근거로 친절한 자연어 답변을 제시해야 한다.
5. 관련 지식이 없으면 답을 지어내지 않고 확인 불가를 안내해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력/저장: 피부과·성형외과 주요 시술의 특징·주의사항 지식 데이터.
- 입력: 신입사원의 자연어 질문.
- 출력: 검색된 지식 기반 답변.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 시술 특징·주의사항 데이터를 사전 입력·저장할 수 있다.
- [ ] 자연어 질문을 입력하면 관련 시술 지식이 검색되어 답변이 나온다.
- [ ] 사전 입력된 시술에 대한 질문에 특징·주의사항이 포함된 답이 반환된다.
- [ ] 데이터에 없는 질문에는 답을 지어내지 않고 정보 없음을 안내한다.
