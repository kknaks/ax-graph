# 제안서 작성 도우미

> 한 줄 정의: 담당자가 회의록·아이디어를 입력하면, AI가 부족한 부분을 인터뷰로 물어보고 기존 우수 제안서 양식을 참고해 새 제안서 파일을 완성해 준다.

## 1. 요구 배경
- **현재 불편(as-is)**: 미팅 기록이나 아이디어만으로는 제안서에 필요한 정보가 부족한데, 어떤 항목이 비었는지 사람이 일일이 파악해야 한다. 원문은 "미팅 기록을 넣었을 때 부족한 부분을 AI가 나한테 물어보고(인터뷰), 예전에 써둔 잘된 제안서 양식을 참고해서 새 제안서 파일로 만들어주면 좋겠습니다"라고 요구한다.
- **필요 이유**: AI가 빠진 정보를 인터뷰로 채우고 우수 양식을 재사용하면, 담당자는 초안 작성 부담 없이 완성된 제안서 파일을 얻는다.

## 2. 기능 정의
- **입력한다**: 회의록·아이디어 텍스트를 제안서 작성 세션의 입력으로 받는다.
- **탐지한다**: 입력에서 제안서 작성에 필요한데 빠져 있는 항목을 식별한다.
- **질문한다**: 부족한 항목을 담당자에게 인터뷰 방식으로 하나씩 되묻고 답변을 수집한다.
- **참고한다**: 과거의 잘된 제안서 양식을 검색해 구조·표현을 참고한다.
- **생성한다**: 수집한 정보와 참고 양식을 결합해 새 제안서 파일로 산출한다.

## 3. 유저 플로우 (반드시 표, 케이스별)
정상 1건 + 예외/대안 최소 1건.

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 회의록을 붙여넣고 제안서 작성을 요청한다 | 부족한 항목을 탐지해 인터뷰 질문을 던지고, 답변과 기존 우수 제안서 양식을 참고해 조립한다 | 완성된 제안서 파일 |
| 예외 | 인터뷰 질문에 답하지 않고 넘어간다 | 미응답 항목을 비워둔 채 표시하고, 참고 양식 기준으로 채울 수 있는 부분만 채운다 | 미응답 항목이 표시된 초안 제안서 파일 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "미팅 기록을 넣었을 때 부족한 부분을 AI가 나한테 물어보고(인터뷰)" — 담당자가 미팅 기록을 붙여넣는다.
- **기대 출력**: AI가 "제안 대상 고객사와 예산 범위가 비어 있습니다"처럼 부족한 항목을 되묻는 인터뷰 질문을 제시한다.
- **입력**: "예전에 써둔 잘된 제안서 양식을 참고해서 새 제안서 파일로 만들어주면 좋겠습니다."
- **기대 출력**: 수집된 답변과 참고 양식을 반영한 새 제안서 파일이 생성된다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 회의록·아이디어 텍스트를 제안서 작성 입력으로 받아야 한다.
2. 입력에서 제안서에 필요한데 빠진 항목을 탐지해야 한다.
3. 부족한 항목을 담당자에게 인터뷰 방식으로 되물어 답변을 수집해야 한다.
4. 과거의 잘된 제안서 양식을 참고해 구조·표현에 반영해야 한다.
5. 수집한 정보와 참고 양식을 결합해 새 제안서 파일로 산출해야 한다.
6. 미응답 항목이 있으면 비워둔 채 표시해 담당자가 확인할 수 있게 해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 회의록·아이디어 텍스트, 인터뷰 답변.
- 저장/참고: 과거 우수 제안서 양식.
- 출력: 완성된 제안서 파일.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 회의록을 입력하면 부족한 항목을 되묻는 인터뷰 질문이 최소 1개 제시된다.
- [ ] 인터뷰 답변이 최종 제안서 내용에 반영된다.
- [ ] 과거 우수 제안서 양식의 구조가 산출 제안서에 참고·반영된다.
- [ ] 인터뷰 완료 후 새 제안서 파일이 산출된다.
- [ ] 미응답 항목이 있으면 제안서에 비워진 채 표시된다.
