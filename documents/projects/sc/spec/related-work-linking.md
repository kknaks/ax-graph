# 연관 업무 연결

> 한 줄 정의: 작업자가 수정·신규 요청을 처리할 때 / 시스템이 이전 작업과 유관 작업을 연결해 주면 / 요청자·작업자·작업 경로를 한자리에서 확인한다.

## 1. 요구 배경
- **현재 불편(as-is)**: 수정 요청이 오면 어떤 이전 작업의 연장인지 손으로 되짚어야 하고, 신규 요청도 유관 작업을 찾기 어렵다. 원문은 "수정 요청 시 이전 작업 연결 기능(요청자, 작업자, 작업 경로 확인 필요)"과 "신규 요청 시 유관 작업 연결 기능(요청자, 작업자, 작업 경로 확인 필요)"을 요구한다.
- **필요 이유**: 요청을 이전·유관 작업에 자동·수동으로 이어 붙여 맥락과 담당·경로를 즉시 확인하게 함으로써 되짚는 수고와 누락을 없앤다.

## 2. 기능 정의
- 수정 요청이 들어오면 그 요청과 이어지는 이전 작업을 자동으로 찾아 연결 후보로 제시한다.
- 신규 요청이 들어오면 내용이 겹치는 유관 작업을 자동으로 찾아 연결 후보로 제시한다.
- 자동 제시된 연결을 작업자가 수동으로 확정·추가·해제한다.
- 연결된 각 작업의 요청자·작업자·작업 경로를 함께 표시한다.
- 하나의 요청에 연결된 작업들을 목록으로 묶어 함께 열람하게 한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 수정 요청을 등록한다 | 관련 이전 작업을 찾아 연결 후보로 제시하고, 사용자가 확정하면 연결을 저장한다 | 이전 작업과 연결된 요청 + 요청자·작업자·작업 경로 표시 |
| 예외 | 신규 요청을 등록했으나 유관 작업이 없다 | 유관 작업을 찾지 못해 후보가 비었음을 알리고 수동 연결 입력창을 연다 | 연결 없이 등록되며, 이후 수동으로 연결 추가 가능 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "수정 요청 시 이전 작업 연결 기능(요청자, 작업자, 작업 경로 확인 필요)"
- **기대 출력**: 수정 요청 화면에 연결된 이전 작업이 표시되고, 그 이전 작업의 요청자·작업자·작업 경로가 함께 보인다.
- **입력**: "신규 요청 시 유관 작업 연결 기능(요청자, 작업자, 작업 경로 확인 필요)"
- **기대 출력**: 신규 요청 등록 시 유관 작업 후보 목록이 뜨고, 선택한 작업이 요청자·작업자·작업 경로와 함께 연결된다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 수정 요청이 등록되면 관련 이전 작업을 자동으로 찾아 연결 후보로 제시해야 한다.
2. 신규 요청이 등록되면 유관 작업을 자동으로 찾아 연결 후보로 제시해야 한다.
3. 자동 제시된 연결을 사용자가 수동으로 확정·추가·해제할 수 있어야 한다.
4. 연결된 각 작업의 요청자·작업자·작업 경로를 함께 표시해야 한다.
5. 유관 작업을 찾지 못한 경우에도 요청 등록을 막지 않고 수동 연결 수단을 제공해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 요청(수정/신규) 내용, 사용자가 지정한 수동 연결.
- 저장: 작업 간 연결 관계, 각 작업의 요청자·작업자·작업 경로.
- 출력: 연결된 작업 목록과 요청자·작업자·작업 경로 표시.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 수정 요청 등록 시 이전 작업이 연결 후보로 자동 제시된다.
- [ ] 신규 요청 등록 시 유관 작업이 연결 후보로 자동 제시된다.
- [ ] 사용자가 연결을 수동으로 추가·해제할 수 있다.
- [ ] 연결된 작업마다 요청자·작업자·작업 경로가 함께 표시된다.
