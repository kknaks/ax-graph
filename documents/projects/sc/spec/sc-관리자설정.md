# 관리자설정

> 한 줄 정의: 관리자가 / 사용자·조직·공통코드 항목을 시스템 화면에서 직접 추가·수정·삭제하면 / 개발업체 요청 없이 운영 기준을 즉시 반영한다.

## 1. 요구 배경
- **현재 불편(as-is)**: 부서·직급·업무 상태·일정 종류 같은 운영 기준을 바꾸려면 그때마다 개발업체에 요청해야 해서 반영이 느리고 번거롭다. 원문은 관리자 화면에서 "개발업체에 매번 요청하지 않고 다음 항목을 직접 설정할 수 있어야 합니다."라고 명시한다.
- **필요 이유**: 운영 기준의 추가·수정·삭제 권한을 관리자에게 넘겨, 조직 변화나 코드 변경을 시스템 안에서 즉시 반영하기 위함이다.

## 2. 기능 정의
- **관리한다** 사용자 계정을 추가·수정·삭제하고 사용자별 권한을 설정한다.
- **관리한다** 부서·파트·TF·자회사·직급 등 조직 구성 항목을 추가·수정한다.
- **관리한다** 업무 상태·일정 종류·계약 종류·소프트웨어 종류·컨택 상태·자료실 카테고리 등 분류 코드를 추가·수정·삭제한다.
- **설정한다** 계약·비자·라이선스 만료 등에 대한 알림 기준을 등록·변경한다.
- **관리한다** 시스템 전반에서 재사용되는 공통코드를 한 화면에서 등록·수정·삭제한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 관리자가 관리자 설정 화면에서 새 일정 종류 항목을 추가한다 | 관리자 설정 접근 권한을 확인하고 항목을 공통코드에 저장한다 | 일정 등록 화면의 일정 종류 선택 목록에 새 항목이 즉시 반영됨 |
| 예외 | 관리자 설정 접근 권한이 없는 사용자가 설정 화면에 진입을 시도한다 | 권한 등급·관리자 설정 접근 권한을 검사해 접근을 차단한다 | 설정 화면이 열리지 않고 접근 불가 안내가 표시됨 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "개발업체에 매번 요청하지 않고 다음 항목을 직접 설정할 수 있어야 합니다." — 관리자가 신규 TF를 생성한다.
- **기대 출력**: 조직 목록에 새 TF가 추가되고, 사용자 권한·업무 공간에서 해당 TF를 선택할 수 있게 된다.
- **입력**: "컨택 상태 관리" — 관리자가 컨택 상태에 새 단계를 추가한다.
- **기대 출력**: 에이전시 컨택 관리의 상태 선택 목록에 새 단계가 나타나고 상태별 필터에도 반영된다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 관리자는 사용자를 추가·수정·삭제하고 사용자별 권한을 설정할 수 있어야 한다.
2. 관리자는 부서·파트를 추가 및 수정하고, TF·자회사를 추가할 수 있어야 한다.
3. 관리자는 직급을 관리할 수 있어야 한다.
4. 관리자는 업무 상태·일정 종류·계약 종류·소프트웨어 종류·컨택 상태·자료실 카테고리를 관리할 수 있어야 한다.
5. 관리자는 알림 기준을 관리할 수 있어야 한다.
6. 관리자는 공통코드를 관리할 수 있어야 한다.
7. 관리자 설정 접근은 별도 권한(관리자 설정 접근)을 가진 사용자만 사용할 수 있어야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력/저장/출력: 사용자·권한 설정값, 조직 구성 항목(부서·파트·TF·자회사·직급), 분류 코드(업무 상태·일정 종류·계약 종류·소프트웨어 종류·컨택 상태·자료실 카테고리), 알림 기준값, 공통코드 항목.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 관리자가 개발업체 요청 없이 사용자·권한을 직접 추가·수정·삭제·설정할 수 있다.
- [ ] 관리자가 부서·파트·TF·자회사·직급을 추가·수정할 수 있다.
- [ ] 관리자가 업무 상태·일정 종류·계약 종류·소프트웨어 종류·컨택 상태·자료실 카테고리·알림 기준·공통코드를 관리할 수 있다.
- [ ] 관리자 설정 접근 권한이 없는 사용자는 설정 화면에 진입할 수 없다.
