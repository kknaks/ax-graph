# 로그인 권한관리

> 한 줄 정의: 직원이 개인 계정으로 로그인하면, 시스템이 조직·메뉴·등급별 접근권한을 판정해 허용된 범위의 기능과 데이터만 제공한다.

## 1. 요구 배경
- **현재 불편(as-is)**: 경영관리부와 관련 TF·자회사의 업무를 하나로 관리해야 하지만 직원마다 볼 수 있어야 하는 조직·메뉴가 다르고, "직원 주민등록번호, 급여, 계좌번호, 계약서 등 민감한 개인정보"가 함께 담기므로 누구나 열람하게 두면 안 된다. 계정·권한을 개발업체에 매번 요청하지 않고 직접 통제할 수단도 없다.
- **필요 이유**: 개인 계정 로그인과 조직·메뉴·등급별 권한 설정을 갖추면 각 직원이 자신에게 허용된 범위만 보게 되어 민감정보 노출과 무단 접근을 막는다.

## 2. 기능 정의
- 직원별 개인 아이디·비밀번호로 로그인시키고, 최초 로그인 시 비밀번호를 직접 설정하게 한다.
- 비밀번호 변경·초기화, 관리자에 의한 사용자 등록·사용중지, 퇴사자 계정 비활성화를 처리한다.
- 일정 시간 미사용 시 자동 로그아웃하고, 로그인 실패 횟수를 제한한다.
- 로그인 및 주요 정보 수정 이력을 저장한다.
- 사용자별로 접근 가능한 조직(부서·파트·TF·자회사)과 메뉴, 권한 등급(최고관리자~조회전용 5등급)과 세부 권한을 설정한다.
- 주민등록번호·급여·계좌번호·계약서 등 민감정보는 별도 권한을 가진 사용자만 열람하도록 판정한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 직원이 아이디·비밀번호로 로그인한다 | 계정을 인증하고, 사용자에게 설정된 조직·메뉴·등급·세부 권한을 판정한다 | 허용된 조직·메뉴만 노출된 화면 진입, 로그인 이력 저장 |
| 정상(최초) | 신규 등록된 직원이 첫 로그인한다 | 비밀번호 직접 설정을 요구하고 이후 인증에 사용한다 | 본인 설정 비밀번호로 계정 활성화 |
| 예외(권한 없음) | 조회전용 사용자가 급여정보 열람을 시도한다 | 해당 사용자에게 급여정보 조회 권한이 없음을 판정한다 | 접근 차단, 민감정보 미표시 |
| 예외(실패 초과) | 사용자가 비밀번호를 반복 오입력한다 | 로그인 실패 횟수 제한에 도달하면 인증을 막는다 | 로그인 차단, 실패 이력 기록 |
| 예외(퇴사자) | 퇴사자가 로그인을 시도한다 | 비활성화된 계정으로 판정한다 | 즉시 접속 차단 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "최초 로그인 시 비밀번호 직접 설정"
  - **기대 출력**: 신규 계정 첫 접속 시 비밀번호 설정 화면을 띄우고, 설정한 값으로만 이후 로그인이 되게 한다.
- **입력**: "직원 주민등록번호, 급여, 계좌번호, 계약서 등 민감한 개인정보는 별도 권한을 가진 사용자만 열람할 수 있도록 요청드립니다."
  - **기대 출력**: 별도 열람 권한이 없는 사용자에게는 해당 항목을 마스킹·차단하고, 권한자에게만 원본을 표시한다.
- **입력**: "일정 시간 미사용 시 자동 로그아웃"
  - **기대 출력**: 설정된 미사용 시간이 지나면 세션을 종료하고 재로그인을 요구한다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 직원별 개인 아이디와 비밀번호로 로그인할 수 있어야 한다.
2. 최초 로그인 시 비밀번호를 직접 설정하게 해야 한다.
3. 비밀번호 변경 및 관리자에 의한 초기화가 가능해야 한다.
4. 관리자가 사용자를 등록·사용중지할 수 있고, 퇴사자 계정을 비활성화할 수 있어야 한다.
5. 일정 시간 미사용 시 자동으로 로그아웃되어야 한다.
6. 로그인 실패 횟수를 제한해야 한다.
7. 로그인 및 주요 정보 수정 이력을 저장해야 한다.
8. 사용자별로 접근 가능한 조직(경영관리부 산하 파트, TF 프로젝트, 자회사)과 메뉴를 설정할 수 있어야 한다.
9. 권한 등급을 최고관리자·일반관리자·파트장·일반사용자·조회전용 사용자 5단계로 구분해 설정할 수 있어야 한다.
10. 메뉴 조회·자료 등록·수정·삭제·엑셀 다운로드·파일 다운로드·직원 개인정보 조회·급여정보 조회·관리자 설정 접근 권한을 각각 설정할 수 있어야 한다.
11. 주민등록번호·급여·계좌번호·계약서 등 민감정보는 별도 권한자만 열람할 수 있어야 한다.
12. 일반사용자는 본인이 권한을 가진 파트와 본인 업무만 볼 수 있어야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 아이디, 비밀번호, 관리자 지정 조직·메뉴·등급·세부 권한 설정값.
- 저장: 계정 정보(비밀번호는 암호화), 사용자별 권한 설정, 로그인 이력, 주요 정보 수정 이력.
- 출력: 인증 결과, 권한에 따라 필터링된 조직·메뉴·데이터 노출.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 신규 계정 최초 로그인 시 비밀번호 설정 화면이 뜨고, 설정 전에는 로그인이 완료되지 않는다.
- [ ] 로그인 실패가 지정 횟수를 넘으면 인증이 차단되고 실패 이력이 남는다.
- [ ] 지정된 미사용 시간이 지나면 세션이 자동 종료된다.
- [ ] 조회전용 사용자에게는 자료 등록·수정·삭제 기능이 노출되지 않는다.
- [ ] 급여정보 열람 권한이 없는 사용자에게 급여·계좌번호 등 민감정보가 표시되지 않는다.
- [ ] 퇴사 처리된 계정으로는 로그인할 수 없다.
- [ ] 로그인 및 주요 정보 수정 이력에서 누가 언제 수행했는지 확인할 수 있다.
