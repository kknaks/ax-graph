# 업무분장

> 한 줄 정의: 관리자·직원이 직원별 담당업무를 항목별로 기입하면, 시스템이 직원·파트·업무구분별로 조회 가능한 업무분장표를 제공하고 엑셀로 출력하며, 요청자는 본인 작업의 담당 디자이너와 진행·예정 상태를 확인한다.

## 1. 요구 배경
- **현재 불편(as-is)**: 누가 어떤 업무를 담당하고 세부업무·업무주기·대체 담당자가 누구인지가 한 곳에 정리되어 있지 않다. 요청서 §16은 "직원별 담당업무를 정리할 수 있는 업무분장표가 필요합니다"라고 명시하며, 담당업무와 관련 파일·매뉴얼이 흩어져 있어 인수인계나 대체 처리 시 파악이 어렵다. 여기에 더해 디자인·영상팀에서는 "기획안 요청 후 본인 작업을 어떤 디자이너가 하는지 확인 기능"이 없어 요청자가 자기 작업의 담당자를 파악하지 못하고, "작업이 들어갔는지 작업 예정인지 확인 할 수 있는 기능(기획안 수정 시 유용)"이 없어 지금 진행 중인지 아직 예정 상태인지 구분하지 못한다.
- **필요 이유**: 직원별 담당업무·세부업무·업무주기·중요도·대체 담당자·관련 파일·매뉴얼을 표 하나로 정리하고 다양한 기준으로 조회·출력하게 하며, 요청자가 본인 작업의 담당 디자이너와 진행·예정 상태를 함께 확인하게 하여 업무 진행·분배 체계를 한눈에 파악하게 한다.

## 2. 기능 정의
- 직원별로 이름·소속·업무 구분·담당업무·세부업무·업무 주기·중요도·대체 담당자·관련 파일·업무 매뉴얼·비고 항목을 등록·수정한다.
- 등록된 업무분장 항목을 직원별로 조회한다.
- 등록된 업무분장 항목을 파트별로 조회한다.
- 등록된 업무분장 항목을 업무구분별로 조회한다.
- 조회한 업무분장표를 엑셀 파일로 출력한다.
- 기획안 요청자에게 본인 작업을 담당하는 디자이너가 누구인지 표시한다.
- 각 작업이 진행 중(작업 들어감)인지 예정 상태인지 구분해 표시한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 관리자가 특정 파트를 선택하고 업무분장 항목을 등록한 뒤 파트별로 조회 후 엑셀 출력을 누른다 | 입력한 항목을 저장하고 선택한 파트 기준으로 필터링해 표로 표시, 표 내용을 엑셀 파일로 생성한다 | 파트별 업무분장표 화면 + 엑셀 파일 |
| 정상(진행·분배 확인) | 기획안을 요청한 사용자가 본인 작업의 담당·진행 상태를 확인한다 | 요청 건에 매핑된 담당 디자이너를 찾고 해당 작업 상태를 진행 중/예정으로 구분해 표시한다 | 담당 디자이너 이름 + 진행/예정 상태가 표시된 확인 화면 |
| 예외 | 담당업무·매뉴얼이 아직 정해지지 않은 직원을 조회한다 | 미입력 항목은 빈 칸으로 두고 조회 결과에 해당 직원 행을 표시한다 | 미입력 칸이 있는 업무분장표(입력 필요 상태 확인) |
| 예외(미배정) | 아직 담당자가 배정되지 않은 기획안의 담당·상태를 확인한다 | 담당 디자이너가 없으면 미배정으로, 작업이 아직 시작 전이면 예정 상태로 표시한다 | 미배정/예정으로 표시된 확인 화면 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "직원별 담당업무를 정리할 수 있는 업무분장표가 필요합니다." — 관리자가 회계파트 직원의 담당업무를 등록.
  - **기대 출력**: 이름·소속·업무 구분·담당업무·세부업무·업무 주기·중요도·대체 담당자가 채워진 회계파트 업무분장 행.
- **입력**: "직원별, 파트별, 업무구분별로 조회할 수 있고 엑셀로 출력할 수 있어야 합니다." — 사용자가 업무구분별 조회 후 엑셀 출력 요청.
  - **기대 출력**: 선택한 업무구분에 해당하는 행만 필터링된 표와 그 내용을 담은 엑셀 파일.
- **입력**: "기획안 요청 후 본인 작업을 어떤 디자이너가 하는지 확인 기능" — 기획안을 올린 요청자가 담당 확인을 요청.
  - **기대 출력**: 해당 요청 건을 담당하는 디자이너 이름과 함께 표시된 확인 화면.
- **입력**: "작업이 들어갔는지 작업 예정인지 확인 할 수 있는 기능(기획안 수정 시 유용)" — 요청자가 작업 상태를 확인.
  - **기대 출력**: 해당 작업이 진행 중인지 예정인지 구분해 표시된 상태 값.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 업무분장 항목으로 이름·소속·업무 구분·담당업무·세부업무·업무 주기·중요도·대체 담당자·관련 파일·업무 매뉴얼·비고를 입력할 수 있어야 한다.
2. 업무분장표를 직원별로 조회할 수 있어야 한다.
3. 업무분장표를 파트별로 조회할 수 있어야 한다.
4. 업무분장표를 업무구분별로 조회할 수 있어야 한다.
5. 조회 결과를 엑셀 파일로 출력할 수 있어야 한다.
6. 대체 담당자를 지정하여 담당자 부재 시 대체 인력을 확인할 수 있어야 한다.
7. 관련 파일과 업무 매뉴얼을 각 담당업무에 연결할 수 있어야 한다.
8. 기획안 요청 건에 대해 본인 작업을 담당하는 디자이너가 누구인지 요청자가 확인할 수 있어야 한다.
9. 각 작업의 상태를 진행 중(작업 들어감)과 예정으로 구분해 확인할 수 있어야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력/저장: 이름, 소속, 업무 구분, 담당업무, 세부업무, 업무 주기, 중요도, 대체 담당자, 관련 파일, 업무 매뉴얼, 비고, 담당 디자이너, 작업 상태(진행/예정).
- 출력: 직원별·파트별·업무구분별 필터링된 업무분장표, 엑셀 파일, 요청 건별 담당 디자이너·진행/예정 상태.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 이름·소속·업무 구분·담당업무·세부업무·업무 주기·중요도·대체 담당자·관련 파일·업무 매뉴얼·비고 11개 항목을 모두 입력·저장할 수 있다.
- [ ] 직원별·파트별·업무구분별 세 가지 기준으로 각각 조회 결과가 필터링되어 표시된다.
- [ ] 조회한 업무분장표를 엑셀 파일로 내려받을 수 있다.
- [ ] 대체 담당자가 지정된 항목에서 대체 인력이 표에 표시된다.
- [ ] 기획안 요청자가 본인 작업의 담당 디자이너 이름을 확인할 수 있다.
- [ ] 각 작업이 진행 중인지 예정인지 구분되어 표시된다.
