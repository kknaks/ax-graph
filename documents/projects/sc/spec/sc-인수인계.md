# 인수인계

> 한 줄 정의: 인계자가 담당업무·진행업무·거래처·계정 권한 등을 항목별로 작성하면, 인수자가 확인 처리하고 인수인계서를 PDF로 내려받는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 퇴사·휴직·담당자 변경 시 인계 내용이 개인별로 흩어져 진행 중 업무·거래처·계정 권한이 누락된 채 넘어간다. 원문은 "퇴사, 휴직 또는 담당자 변경 시 사용할 수 있는 인수인계 관리 기능이 필요합니다"라고 요구한다.
- **필요 이유**: 정해진 항목으로 인계 내용을 한 문서에 모아 작성·확인·보관함으로써 담당자 교체 시 업무 공백을 없앤다.

## 2. 기능 정의
- 인계자·인수자·인수인계 기간을 지정해 인수인계서를 생성한다.
- 담당업무·진행 중 업무·정기 업무·주요 거래처·주요 연락처·사용 시스템·파일 및 문서 위치·미결사항·주의사항을 항목별로 입력받는다.
- 계정 및 권한 이전 여부를 기록하되 비밀번호는 문서에 직접 노출하지 않는다.
- 인수자의 확인 서명 또는 확인 처리로 인계 완료 상태를 확정한다.
- 완성된 인수인계서를 PDF로 다운로드한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 인계자가 담당업무·진행업무·거래처·계정 권한 이전 여부·미결사항을 입력하고 인수자를 지정한다 | 항목을 인수인계서로 저장하고 인수자에게 확인 요청을 보낸다 | 인수자 확인 처리된 인수인계서 + PDF 다운로드 |
| 예외 | 인계 항목에 시스템 계정 비밀번호를 기입하려 한다 | 비밀번호는 문서에 노출하지 않고 별도 보안 절차 전달로 안내한다 | 비밀번호가 빠진 인수인계서(별도 전달 표시) |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "계정 및 권한 이전 여부" 항목에 인계 대상 시스템과 이전 여부를 표시.
  - **기대 출력**: 계정 이전 여부가 기록되되 비밀번호는 문서에 표시되지 않고 별도 보안 절차로 전달됨을 안내.
- **입력**: "인수인계서를 PDF로 다운로드할 수 있도록 요청드립니다."
  - **기대 출력**: 작성·확인 처리된 인수인계서를 PDF 파일로 내려받음.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 인계자·인수자·인수인계 기간을 지정해 인수인계서를 생성할 수 있어야 한다.
2. 담당업무·진행 중 업무·정기 업무·주요 거래처·주요 연락처·사용 시스템·파일 및 문서 위치·미결사항·주의사항·첨부파일을 항목별로 입력할 수 있어야 한다.
3. 계정 및 권한 이전 여부를 기록하되 비밀번호는 인수인계서에 직접 노출하지 않아야 한다.
4. 비밀번호는 별도의 보안 절차로 전달할 수 있게 해야 한다.
5. 인수자가 확인 서명 또는 확인 처리를 할 수 있어야 한다.
6. 완성된 인수인계서를 PDF로 다운로드할 수 있어야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 인계자/인수자, 인수인계 기간, 담당·진행·정기 업무, 거래처·연락처, 사용 시스템, 계정·권한 이전 여부, 파일·문서 위치, 미결사항, 주의사항, 첨부파일.
- 저장: 인수인계서 레코드, 확인 처리 상태.
- 출력: PDF 인수인계서(비밀번호 제외).

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 인계자·인수자·기간과 모든 인계 항목을 입력해 인수인계서를 저장할 수 있다.
- [ ] 인수자가 확인 서명 또는 확인 처리를 하면 인계 완료 상태로 표시된다.
- [ ] 인수인계서를 PDF로 다운로드할 수 있다.
- [ ] 비밀번호가 인수인계서 본문·PDF에 노출되지 않고 별도 보안 절차로 전달된다.
