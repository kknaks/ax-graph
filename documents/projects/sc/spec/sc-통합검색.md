# 통합검색

> 한 줄 정의: 사용자가 시스템 상단 검색창에 한 번 질의하면, 자신이 권한을 가진 업무·회의록·직원·계약·자료실·메모·컨택·레퍼런스·업무보고서·인수인계서를 아울러 찾아 결과를 얻는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 업무·회의록·직원·계약 등 자료가 메뉴별로 흩어져 있어 원하는 정보를 찾으려면 메뉴를 일일이 옮겨 다니며 각각 검색해야 한다. 요청서는 "시스템 상단에 통합검색 기능이 필요합니다"라고 명시한다.
- **필요 이유**: 상단 한 곳에서 전 메뉴 자료를 한 번에 검색해 자료를 찾는 시간을 줄인다.

## 2. 기능 정의
- 시스템 상단 고정 검색창에서 입력한 검색어를 받는다.
- 업무·회의록·직원·계약·자료실·메모·컨택 대상·레퍼런스·업무보고서·인수인계서를 검색 대상으로 조회한다.
- 검색 결과를 대상 종류(메뉴)별로 구분해 표시하고 원문 항목으로 바로 이동시킨다.
- 검색 시점에 사용자의 조직·메뉴·자료 접근권한을 적용해 권한 없는 자료를 결과에서 제외한다.

## 3. 유저 플로우 (반드시 표, 케이스별)
정상 1건 + 예외/대안 최소 1건.

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 상단 검색창에 키워드를 입력하고 검색한다 | 10개 검색 대상을 조회하고 사용자 권한으로 필터링해 대상별로 묶는다 | 메뉴별로 구분된 검색 결과 목록, 각 항목 클릭 시 원문 이동 |
| 예외(권한 없음) | 권한 없는 파트의 계약을 포함할 수 있는 키워드로 검색한다 | 권한 검사에서 접근 불가 자료를 결과 집합에서 제외한다 | 권한 있는 자료만 표시되고, 권한 없는 계약은 결과에 나타나지 않음 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "업무, 회의록, 직원, 계약, 자료실, 메모, 컨택 대상, 레퍼런스, 업무보고서, 인수인계서" 중 하나에 해당하는 이름/키워드를 상단 검색창에 입력.
  - **기대 출력**: 해당 키워드를 포함하는 항목들이 대상 종류별로 묶여 나타나고, 항목 선택 시 원문으로 이동한다.
- **입력**: 급여정보·계약서 등 열람 권한이 없는 사용자가 해당 계약명을 검색.
  - **기대 출력**: "권한이 없는 자료는 검색 결과에도 표시되지 않아야 합니다"에 따라 그 계약이 결과에 나타나지 않는다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 검색창은 시스템 상단에 항상 노출되어야 한다.
2. 검색 대상은 업무·회의록·직원·계약·자료실·메모·컨택 대상·레퍼런스·업무보고서·인수인계서를 포함해야 한다.
3. 검색 결과는 어느 대상(메뉴)에서 나온 항목인지 구분되어야 한다.
4. 각 검색 결과 항목은 해당 원문 화면으로 이동할 수 있어야 한다.
5. 검색은 조회 시점의 사용자 조직·메뉴·자료 접근권한을 적용해, 권한이 없는 자료는 검색 결과에도 표시하지 않아야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 검색어(문자열), 검색 실행 사용자와 그 권한 정보.
- 조회 대상: 업무·회의록·직원·계약·자료실·메모·컨택·레퍼런스·업무보고서·인수인계서 항목.
- 출력: 권한 필터링을 거친 대상별 검색 결과 목록.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 상단 검색창에서 검색하면 10개 대상(업무·회의록·직원·계약·자료실·메모·컨택·레퍼런스·업무보고서·인수인계서)에서 결과가 조회된다.
- [ ] 검색 결과가 대상 종류별로 구분되어 표시되고, 항목 선택 시 원문으로 이동한다.
- [ ] 권한이 없는 자료는 검색 결과에 표시되지 않는다.
- [ ] 동일 키워드라도 권한이 다른 두 사용자의 검색 결과 집합이 권한 범위에 따라 달라진다.
