# 파일 데이터관리

> 한 줄 정의: 사용자가 시스템 곳곳에서 파일을 올리고 내려받고 지우면, 시스템이 확장자·용량 제한과 미리보기·다운로드 이력·휴지통·백업으로 그 파일과 데이터를 안전하게 보관·복구·내보낸다.

## 1. 요구 배경
- **현재 불편(as-is)**: 여러 메뉴(자료실·직원 문서·계약·소프트웨어 등)에서 파일을 다루지만 업로드 제한·미리보기·삭제 복구·백업 같은 공통 처리가 표준화되어 있지 않다. 실수로 지운 자료를 되살릴 방법이 없고, 데이터·백업 관리도 개발업체에 매번 의존해야 한다.
- **필요 이유**: 파일·데이터 처리를 공통 기능으로 묶어 업로드 제한·미리보기·삭제 복구·엑셀 내보내기·백업을 시스템이 일관되게 제공하면, 자료 유실 위험과 운영 부담을 함께 줄인다.

## 2. 기능 정의
- 파일 업로드 시 확장자와 용량이 관리자가 설정한 제한을 만족하는지 검사한다.
- 이미지와 PDF 파일은 다운로드 없이 화면에서 미리보기로 보여준다.
- 여러 파일을 한 번에 업로드하고, 등록된 파일을 교체·삭제할 수 있게 한다.
- 파일 다운로드가 발생하면 누가 언제 어떤 파일을 받았는지 이력으로 기록한다.
- 삭제한 자료를 휴지통에 일정 기간 보관해 복구할 수 있게 하고, 관리자가 정기 자동 백업과 수동 백업, 데이터 엑셀 내보내기를 수행한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 사용자가 이미지·PDF 등 파일 여러 개를 선택해 업로드한다 | 확장자·용량 제한을 검사하고 통과분을 저장, 이미지·PDF는 미리보기를 생성한다 | 업로드된 파일 목록과 미리보기가 표시된다 |
| 예외(제한 초과) | 사용자가 허용되지 않은 확장자나 용량 초과 파일을 올린다 | 제한 위반을 감지해 해당 파일을 저장하지 않고 사유를 안내한다 | 거부 안내 메시지, 나머지 정상 파일만 업로드 |
| 예외(삭제 후 복구) | 사용자가 자료를 삭제하고, 이후 관리자가 복구를 요청한다 | 삭제 자료를 휴지통에 일정 기간 보관하다가 관리자 복구 시 원위치로 되돌린다 | 복구된 자료가 원래 위치에 다시 표시된다 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "직원이 삭제한 자료도 관리자가 일정 기간 복구할 수 있도록 요청드립니다."
  - **기대 출력**: 직원이 지운 자료가 즉시 영구 삭제되지 않고 휴지통에 보존기간 동안 남아, 관리자가 목록에서 선택해 원위치로 복구한다.
- **입력**: "정기 자동 백업" / "관리자 데이터 백업"
  - **기대 출력**: 관리자가 필요 시 수동 백업을 실행하고, 시스템이 설정된 주기에 따라 데이터를 자동으로 백업본으로 남긴다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 관리자가 업로드 허용 확장자와 파일 용량 제한을 설정할 수 있어야 한다.
2. 업로드 시 확장자·용량 제한을 위반하는 파일은 거부하고 사유를 안내해야 한다.
3. 이미지와 PDF 파일은 화면에서 미리보기로 표시해야 한다.
4. 다중 파일 업로드, 파일 교체, 파일 삭제를 지원해야 한다.
5. 파일 다운로드 이력(누가·언제·어떤 파일)을 기록해야 한다.
6. 삭제된 자료는 휴지통에 일정 기간 보관되고, 관리자가 복구할 수 있어야 한다.
7. 직원이 삭제한 자료도 관리자가 보존기간 내에 복구할 수 있어야 한다.
8. 데이터를 엑셀로 내보낼 수 있어야 한다.
9. 관리자가 수동으로 데이터 백업을 실행할 수 있고, 시스템이 정기 자동 백업을 수행해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 업로드 파일(확장자·용량), 삭제 요청, 백업 실행 요청.
- 저장: 원본 파일, 휴지통 보관 자료, 다운로드 이력, 백업본.
- 출력: 미리보기, 엑셀 내보내기 파일, 복구된 자료, 백업본.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 허용되지 않은 확장자나 용량 초과 파일을 업로드하면 저장되지 않고 사유가 표시된다.
- [ ] 이미지·PDF 파일을 다운로드하지 않고 화면 미리보기로 확인할 수 있다.
- [ ] 여러 파일을 한 번에 업로드하고, 등록된 파일을 교체·삭제할 수 있다.
- [ ] 파일 다운로드 시 누가 언제 받았는지 이력에 기록된다.
- [ ] 직원이 삭제한 자료가 휴지통에 보존기간 동안 남고, 관리자가 복구하면 원위치로 되돌아온다.
- [ ] 관리자가 데이터를 엑셀로 내보내고, 수동 및 정기 자동 백업을 확인할 수 있다.
