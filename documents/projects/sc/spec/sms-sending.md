# 문자 발송

> 한 줄 정의: 영업·업무 담당자가 프로그램 안에서 저장된 문자 양식을 골라 발송하면, 외부 문자 발송 서비스를 통해 상대방에게 문자가 전송된다.

## 1. 요구 배경
- **현재 불편(as-is)**: 영업·업무 연락용 문자를 보내려면 프로그램 밖의 별도 문자 발송 도구로 옮겨가야 하고, 자주 쓰는 문안을 매번 새로 작성해야 한다. 원문은 "영업이나 업무 연락용 문자를 이 프로그램 안에서 바로 보내는 기능"을 요구한다.
- **필요 이유**: 프로그램 안에서 양식 선택과 발송을 끝내면 도구 전환과 반복 작성 없이 바로 연락할 수 있다.

## 2. 기능 정의
- 자주 쓰는 문자 양식을 저장한다.
- 저장된 양식을 목록에서 골라 발송 내용에 불러온다.
- 수신자와 문안을 확정해 발송을 요청한다.
- 외부 문자 발송 서비스와 연동해 실제 문자를 전송한다.
- 발송 성공·실패 결과를 사용자에게 돌려준다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | 저장된 양식을 고르고 수신자를 입력해 발송 버튼을 누른다 | 양식을 문안에 채우고 외부 문자 발송 서비스에 전송을 요청한다 | 발송 완료 및 성공 표시 |
| 예외 | 외부 서비스 연동이 끊긴 상태에서 발송을 누른다 | 외부 서비스 응답 실패를 감지한다 | 발송 실패 안내 및 재시도 안내 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "자주 쓰는 문자 양식을 저장해 두고" — 반복 사용할 문안을 양식으로 저장 요청.
  - **기대 출력**: 저장된 양식이 목록에 남고, 다음 발송 시 그 양식을 바로 불러올 수 있다.
- **입력**: "외부 문자 발송 서비스와 연결해서 바로 문자를 보낼 수 있게 해주세요"
  - **기대 출력**: 프로그램 안에서 발송을 요청하면 외부 문자 발송 서비스를 거쳐 실제 문자가 전송된다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 자주 쓰는 문자 양식을 저장하고 다시 불러올 수 있어야 한다.
2. 저장된 양식을 골라 발송 문안으로 채울 수 있어야 한다.
3. 외부 문자 발송 서비스와 연동해 프로그램 안에서 바로 문자를 발송할 수 있어야 한다.
4. 발송 성공·실패 결과를 사용자에게 알려야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 수신자 정보, 발송 문안.
- 저장: 자주 쓰는 문자 양식.
- 출력: 외부 문자 발송 서비스로 전송된 문자, 발송 결과 상태.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 문자 양식을 저장하고 목록에서 다시 불러올 수 있다.
- [ ] 저장된 양식을 선택하면 발송 문안에 그대로 채워진다.
- [ ] 프로그램 안에서 발송을 요청하면 외부 문자 발송 서비스를 통해 문자가 전송된다.
- [ ] 발송 성공 또는 실패 결과가 사용자에게 표시된다.
