# 미션 일정 관리

> 한 줄 정의: 관리자가 대략적 지시와 날짜만 입력하면 / 시스템이 해당 직원 일정에 미션을 등록하고 / 잊지 않도록 알림을 받는다.

## 1. 요구 배경
- **현재 불편(as-is)**: 갑자기 생긴 지시사항이나 미션이 별도 등록 없이 구두로만 전달되어 담당자가 깜빡하고 놓치는 일이 생긴다. 원문은 "갑자기 생긴 지시사항이나 미션이 깜빡하고 묻히지 않도록 챙겨주는 알림 서비스"를 원한다고 밝힌다.
- **필요 이유**: 대략적 지시와 날짜만으로 담당자 일정에 자동 등록하고 반복 알림을 보내면 미션이 묻히지 않는다.

## 2. 기능 정의
- 대략적 지시 문장과 날짜 입력을 받아 담당자와 미션 내용을 식별한다.
- 식별한 미션을 해당 직원의 일정에 자동 등록한다.
- 등록된 미션에 대해 완료 전까지 중간중간 반복 알림을 발송한다.
- 담당자를 특정하지 못하면 등록을 보류하고 입력자에게 재확인을 요청한다.

## 3. 유저 플로우 (반드시 표, 케이스별)

| 케이스 | 행동 (사용자가 한다) | 과정 (시스템이 처리) | 결과 (산출물) |
|---|---|---|---|
| 정상 | "A담당자한테 경쟁사 조사하라고 해줘"와 마감 날짜를 입력한다 | 담당자(A)와 미션(경쟁사 조사)·기한을 파싱해 A의 일정에 등록하고 알림 스케줄을 건다 | A 일정에 등록된 미션 + 예정된 반복 알림 |
| 예외 | 담당자를 명시하지 않은 지시를 입력한다 | 담당자를 특정하지 못해 등록을 보류하고 입력자에게 대상 직원을 되묻는다 | 미등록 상태 + 재확인 요청 메시지 |

## 4. 예시 (최소 2개, 실제 문장 인용)
- **입력**: "A담당자한테 경쟁사 조사하라고 해줘"
- **기대 출력**: A담당자 일정에 "경쟁사 조사" 미션이 지정 날짜로 등록되고, 완료 전까지 반복 알림이 발송된다.
- **입력**: "갑자기 생긴 지시사항이나 미션이 깜빡하고 묻히지 않도록" 챙겨달라는 요구
- **기대 출력**: 등록된 각 미션에 대해 기한 전 중간중간 알림을 보내 누락을 방지한다.

## 5. 상세 요구 (번호 목록, "~해야 한다")
1. 대략적인 지시 문장과 날짜만 입력해도 담당자·미션 내용·기한을 추출해야 한다.
2. 추출한 미션을 해당 직원의 일정에 자동으로 등록해야 한다.
3. 미션이 완료되기 전까지 중간중간 잊지 않도록 알림을 발송해야 한다.
4. 담당자나 필수 항목을 특정할 수 없으면 등록을 보류하고 입력자에게 재확인을 요청해야 한다.

## 6. 다루는 데이터 (해당 시)
- 입력: 지시 문장, 대상 담당자, 마감 날짜.
- 저장: 등록된 미션(담당자·내용·기한)과 알림 스케줄.
- 출력: 일정 등록 결과와 반복 알림 메시지.

## 7. 수용 기준 (체크박스 최소 3개, 참/거짓 판정 가능)
- [ ] 대략적 지시와 날짜만 입력하면 담당자·미션·기한이 파싱되어 일정에 등록된다.
- [ ] 등록된 미션에 대해 기한 전 최소 1회 이상 알림이 발송된다.
- [ ] 담당자를 특정하지 못한 입력은 등록되지 않고 재확인 요청이 반환된다.
