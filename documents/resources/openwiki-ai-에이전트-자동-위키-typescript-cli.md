# OpenWiki: AI 에이전트용 자동 위키 구축 TypeScript CLI 도구

## 요약

[[llm-wiki]] 철학을 TypeScript CLI로 구현한 도구로, 코드 저장소나 개인 지식을 AI 에이전트가 참조하는 위키로 자동 구축·유지한다. SHA 기반 스냅샷 캐시로 CI 무한 루프를 방지하고, 문서 전용 쓰기 가드로 에이전트의 동작 범위를 저장소 안으로 제한하는 것이 기술적 핵심이다.

## 핵심 내용

### 개요 및 핵심 철학

OpenWiki는 흩어진 데이터와 [[에이전틱-ai]] 에이전트를 연결하는 자동 문서화 도구다. 로컬 지식이나 Git 리포지토리를 수집해 통합된 위키로 합성하며, 기존 개발 워크플로우에 자연스럽게 통합되도록 설계됐다.

핵심 철학은 **"직접 위키를 수정하지 말라 — 소스만 건드리면 문서 업데이트는 AI가 전담한다"**는 것이다. 사람이 위키를 직접 편집하는 대신 소스를 관리하면, AI가 문서를 최신 상태로 유지하는 역할을 맡는다. 대상 독자는 사람이 아니라 AI 에이전트다.

### 두 가지 운영 모드

**개인(Brain) 모드**: 명령어 하나로 개인용 로컬 브레인을 구축한다. 흩어진 디지털 조각을 모아 컴팩트한 마크다운 파일로 생성하며, 미해결 질문·최근 트렌드·할 일을 자동 추적·정리한다.

**코드(Code) 모드**: 특정 코드베이스 전담 테크니컬 라이터로 동작한다. 리포지토리 최상단에 지침 파일을 생성해 다른 AI 어시스턴트가 해당 프로젝트에 투입될 때 최신 컨텍스트 길잡이 역할을 수행한다.

두 모드는 독립 트랙으로 운영된다.

### 데이터 수집 커넥터

지원 소스: 로컬 Git 리포지토리(기본), Notion, Gmail, Slack, X(트위터), 해커뉴스 피드, Tavily API 웹 검색.

보안 처리 방식: 모든 인증 흐름은 로컬 환경에서 관리하며, 동적 클라이언트 등록·로컬 콜백 방식을 사용한다. 비밀 키는 환경 설정 파일에 저장하며 외부로 노출되지 않는다.

### 에이전트 워크플로우와 CI 무한 루프 방지

에이전트 실행은 4단계로 구성된다: ①Git 메타데이터 수집 및 문서 상태 스냅샷 생성 → ②에이전트 실행 및 문서 업데이트 → ③변경 내용 확인 → ④메타데이터 기록.

**SHA-256 캐시**가 전체 문서 트리 스냅샷을 유지한다. LLM이 실제로 문서 내용을 변경했을 때만 메타데이터를 갱신하고, 내용 변경이 없으면 커밋을 생성하지 않는다. 이 메커니즘으로 GitHub Actions 등 CI 환경의 무한 커밋 루프를 차단한다.

에이전트 실행 전 호스트 프로세스가 현재 상태·변경 범위·커밋 내역 등 팩트를 사전 수집하고, AI는 정리된 근거를 바탕으로 수정 위치를 판단한다. 이를 통해 헛발질과 연산 낭비를 줄인다.

### 내부 아키텍처 — LLM 유연성과 보안 엔진

**지원 LLM 공급자**: Anthropic, OpenAI, OpenRouter 등 다양한 공급자를 지원하며, API 키 방식 외에 Codex 백엔드를 통한 일반 ChatGPT 계정 로그인 인증도 지원한다. LiteLLM 사용자를 위한 OpenAI 호환 커스텀 URL 옵션도 제공한다.

**보안 엔진**: 딥에이전트 로컬 셸 백엔드 기반의 고도로 제어된 실행 환경을 사용한다. **문서 전용 쓰기 가드**가 에이전트가 리포지토리 밖으로 벗어나는 것을 엄격히 차단한다.

### 시사점

발표자는 마무리 질문으로 이 도구의 함의를 열어둔다: "AI가 리포지토리 컨텍스트와 개인 기억을 스스로 완벽히 유지할 수 있다면, 미래 AI가 할 수 없는 일의 한계는 어디까지인가?"
