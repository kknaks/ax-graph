# PARA 노트 정리법 — 실행도 기준 4분류(티아고 포르테)와 노션 템플릿 활용

## 요약

티아고 포르테가 고안한 PARA는 기존 주제별 노트 분류를 버리고 '실행도(actionability)'를 기준으로 모든 노트를 Project·Area·Resource·Archive 네 폴더에만 넣는 지식관리 방법론이다. 이 영상은 PARA의 원리와 함께, 저자가 직접 제작한 노션 템플릿(인박스, 다중 폴더 연결, D-day, 회고 기능)의 실사용을 시연한다.

## 핵심 내용

### 기존 노트 정리법의 문제

사람들이 흔히 시작하는 **주제별 폴더·태그** 방식은 노트가 쌓일수록 검색과 폴더 관리가 벅차지고, 결국 앱을 포기하게 만든다. 저자는 이 문제의 원인을 앱이 아니라 **정리 방법** 자체로 진단한다. 정리 기준에는 언제나 의도가 내재되어 있고, 그 의도에 의한 결과가 만들어질 수밖에 없다는 것이 출발점이다.

### [[para-method]] — PARA란 무엇인가

PARA는 티아고 포르테가 생산성 컨설팅 노하우를 바탕으로 개발해 2022년 처음 공개한 노트 분류 체계이며, 저서 《세컨드 브레인(Building a Second Brain)》의 저자가 만들었다. 어떤 노트·파일링 시스템에도 적용 가능하고, **폴더 4개**만으로 모든 노트를 분류한다. 핵심 차별점은 정리 기준이 '주제'가 아닌 **'실행도(actionability)'**라는 점이다. 상세 개념은 [[para-method]]에 위임한다.

**냉장고 비유**: 주제별 정리는 채소·육류·과일로 분류하는 방식으로 재고 파악에 유리하고, PARA의 실행도 정리는 김치찌개 재료끼리·불고기 재료끼리 묶는 밀키트 방식이다 — 아웃풋 생산에 직접 연결된다.

### PARA의 네 가지 분류 (실행도 높은 순)

**1. Project (프로젝트)**
끝이 있는 일 — 구체적 달성 목표와 종료 날짜가 존재한다. 현재 전념 중인 가장 중요한 일들이다(자격증 공부, 블로그 글 작성, 제품 런칭, 이사하기 등).

자주 하는 실수: 이사 프로젝트를 진행 중일 때 '부동산 계약 시 주의사항' 노트를 주제 폴더(부동산)에 넣는 것. PARA에서는 **현재 진행 중인 프로젝트 폴더가 우선**이다.

프로젝트 완료 후 처리: 완료된 프로젝트 안의 노트를 관련 다른 프로젝트·영역·자원으로 이동하고, 어디에도 속하지 않으면 아카이브로 보낸다. 노트는 실행도에 따라 살아 움직이는 시스템이 된다.

**2. Area (영역)**
끝이 없는 역할 또는 책임에 해당한다(직업, 건강, 경제 계획, 가족). 프로젝트와의 가장 명확한 구분점은 '끝이 없다'는 것이다. 대부분의 프로젝트는 어느 영역을 발전시키기 위해 생겨난다.

**3. Resource (자원)**
프로젝트·영역에 해당하지 않지만 관심 있는 주제들. 영역과의 차이는 **책임의 유무**다 — 자원 항목은 못하거나 안 해도 별 문제 없다. 동일한 주제라도 사람에 따라 분류가 달라진다(작가에게 '글쓰기'=영역, 일반인에게는 자원).

한국 사용자의 흔한 오해: '자원'이라는 이름 때문에 참고자료·논문을 모으는 곳으로 착각한다. 실제로는 참고자료도 실행도 기준으로 분류한다 — 현재 프로젝트에 유용하면 프로젝트 폴더로.

**4. Archive (아카이브)**
위 세 가지 어디에도 속하지 않는 것들. 실행도 = 0. 완료된 프로젝트·중단된 프로젝트도 포함된다. **버려진 창고가 아니다** — 과거 성공한 보고서·기획서나 지난 여행 준비물 목록처럼 과거 작업물을 새 작업에 레버리지하는 원동력이 된다.

### PARA의 강점 — 세컨드 브레인

PARA 순서대로 정리된 대시보드를 보면 현재 집중하는 중요한 일(프로젝트), 나의 책임 영역, 나의 관심사(자원), 과거 기록(아카이브)을 모두 한눈에 파악할 수 있다. 시간이 지날수록 노트 모음이 **나 자체를 담게 된다**. 이것이 '세컨드 브레인(Second Brain)' — 외부에 위치한 나의 두 번째 뇌 — 개념의 핵심이다.

### 노션 PARA 템플릿 사용법

영상 저자가 직접 제작한 노션 템플릿의 실사용을 시연한다.

**기본 구조**: **인박스(수집함)**에 정리되지 않은 새 노트가 모이고, 노트가 프로젝트·영역·자원으로 배정되면 수집함에서 사라진다. 주기적으로 수집함을 비워주는 것이 핵심 루틴이다.

**노션의 장점 vs. 일반 폴더 방식**:
- 일반 폴더(윈도우·옵시디언): 폴더를 직접 찾아 노트를 복사·붙여넣기 필요
- 노션: 데이터베이스 + 검색·선택 방식으로 훨씬 편리
- **하나의 노트가 여러 폴더에 동시 소속 가능** (예: 여행 프로젝트 + 여행 자원 모두에서 같은 노트 접근)
- **노트 위치 이력 보존**: 폴더 이동 시 위치 정보 유실 문제를 해결 — 과거에 어떤 프로젝트에서 그 노트를 썼는지 확인 가능

**노트 페이지 기능 (4가지)**:
1. 영역·자원 선택 — 선택 시 수집함에서 사라지고 해당 페이지에 표시, 복수 선택 가능
2. 고정하기 — 홈화면 피드에 즐겨찾는 노트 고정
3. 연결된 프로젝트 — 연결 개수 표시, 프로젝트 추가·해제
4. 연결된 노트 — 관련 노트끼리 연결해 지식 확장

**프로젝트 페이지 기능 (5가지)**:
1. 상태 관리: 진행 중 / 시작 안 함 / 완료 / 중단 — 진행 중이 아니면 자동 아카이브
2. 영역·자원 연결: 해당 영역 페이지에서 관련 프로젝트 한눈에 파악
3. 시작일·종료일: 홈화면에서 D-day 표시, 종료일 미설정 시 알림
4. 완료하기: 상태를 완료로 변경 + 회고 노트 자동 생성(필수 아니나 권장)
5. 연결된 노트 목록: 하단에 표시, 새 노트 추가 시 프로젝트 자동 연결

**영역·자원·아카이브**: 동일한 데이터베이스에서 상태 변경으로 전환. 영역 페이지에서 프로젝트를 드래그 앤 드롭으로 상태 변경 가능. 아카이브 항목은 시작 안 한 프로젝트·중단된 프로젝트·완료된 프로젝트·아카이브 처리된 영역·자원을 포함한다.

### 세컨드 브레인 OS와의 관계

이 PARA 노트 템플릿은 세컨드 브레인의 **미니 버전**이다. 세컨드 브레인 OS = PARA + GTD 할일 관리 시스템 추가 → 인생 전체를 하나의 노션 템플릿에서 관리. 기존 세컨드 브레인 OS 데이터를 PARA 노트 템플릿으로 이전할 수 있으며, 이전 방법은 프로젝트·영역·자원·노트 순서로 각 데이터베이스를 전체 선택 후 대상 템플릿의 데이터베이스로 옮기기를 반복하는 방식이다.
