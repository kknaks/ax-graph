## OpenWiki 개요

- **OpenWiki**는 흩어진 데이터와 AI 에이전트를 연결하는 자동 문서화 도구
- 로컬 지식이나 Git 리포지토리를 수집해 하나의 통합된 위키로 묶어 줌
- 터미널에 상주하며 파편화된 데이터를 연결된 결과물로 합성
- 기존 개발 워크플로우에 자연스럽게 통합되도록 설계
- **핵심 대상**: 사람이 아닌 AI 에이전트가 참조하는 위키

## 두 가지 운영 모드

### 개인(Brain) 모드
- 명령어 하나로 개인용 로컬 브레인 구축
- 흩어진 디지털 조각을 모아 컴팩트한 마크다운 파일 생성
- 미해결 질문, 최근 트렌드, 할 일 등을 자동 추적·정리

### 코드(Code) 모드
- 특정 코드베이스 전담 테크니컬 라이터로 동작
- 리포지토리 최상단에 지침 파일 생성
- 다른 AI 어시스턴트가 프로젝트에 투입될 때 최신 컨텍스트 길잡이 역할

> 핵심 마인드셋: "직접 위키를 수정하지 말라. 소스 코드만 건드리면 문서 업데이트는 AI가 전담한다."

## 데이터 수집 커넥터

지원 소스:
- **로컬 Git 리포지토리** (기본)
- **Notion**, **Gmail**, **Slack**
- **X(트위터)**, **해커뉴스 피드**
- **Tavily API**를 통한 웹 검색

### 보안 처리
- 모든 인증 흐름은 로컬 환경에서 관리
- **동적 클라이언트 등록**, **로컬 콜백** 방식 사용
- 비밀 키는 환경 설정 파일에 저장 — 외부 노출 없음

## 에이전트 워크플로우 (4단계)

1. Git 메타데이터 수집 및 현재 문서 상태 스냅샷 생성
2. 에이전트 실행 → 문서 업데이트
3. 변경 내용 확인
4. 메타데이터 기록

### CI 무한 루프 방지 메커니즘
- **SHA-256 캐시**로 전체 문서 트리 스냅샷을 유지
- LLM이 실제로 문서 내용을 변경했을 때만 메타데이터 갱신
- 내용 변경 없으면 커밋 생성 안 함 → GitHub Actions 등 CI 환경의 무한 루프 차단

### 에이전트 정확도 향상
- 에이전트 실행 전 호스트 프로세스가 현재 상태·변경 범위·커밋 내역 등 팩트를 사전 수집
- AI는 정리된 근거를 바탕으로 수정 위치를 판단 → 헛발질 및 연산 낭비 감소

## 내부 아키텍처 및 LLM 유연성

### 지원 LLM 공급자
- **Anthropic**, **OpenAI**, **OpenRouter** 등 다양한 공급자 지원
- API 키 방식 외에 **Codex 백엔드**를 통한 일반 ChatGPT 계정 로그인 인증도 지원
- **LiteLLM** 사용자를 위한 OpenAI 호환 커스텀 URL 옵션 제공

### 보안 엔진
- **딥에이전트 로컬 셸 백엔드** 기반의 고도로 제어된 실행 환경
- **문서 전용 쓰기 가드**: 에이전트가 리포지토리 밖으로 벗어나는 것을 엄격히 차단

## 결론 및 시사점

- 문서 관리를 완전 자동화해 개발자는 소스 코드에만 집중 가능
- 개발자와 AI 에이전트 모두가 참조하기 좋은 체계적 문서 저장소 제공
- 발표자 마무리 질문: "AI가 리포지토리 컨텍스트와 개인 기억을 스스로 완벽히 유지할 수 있다면, 미래 AI가 할 수 없는 일의 한계는 어디까지인가?"
