## 기존 노트 정리법의 문제

- 대부분의 사람들은 처음 메모 앱을 사용할 때 **주제별 폴더·태그** 방식으로 시작
- 노트가 쌓일수록 검색·찾기가 어려워지고, 폴더 관리가 벅차짐
- 결국 노트를 방치하고 앱을 포기하게 됨
- 저자의 진단: 앱의 문제가 아니라 **정리 방법의 문제**

## PARA란?

- **티아고 포르테(Tiago Forte)**가 만든 노트 분류 체계 — 저서 《세컨드 브레인(Building a Second Brain)》의 저자
- 생산성 컨설팅 과정에서 얻은 노하우를 바탕으로 개발, 2022년 처음 공개
- 어떤 노트·파일링 시스템에도 적용 가능하며, **폴더 4개**만으로 모든 노트를 분류
- **핵심 차별점**: 정리 기준이 '주제'가 아닌 **'실행도(actionability)'**

> "정리 기준에는 언제나 그 의도가 내재되어 있고, 그 의도에 의한 결과가 만들어질 수밖에 없다."

### 냉장고 비유
- 주제별 정리 = 채소·육류·과일로 분류 → 재고 파악에 유리
- 실행도 정리 = 김치찌개 재료끼리, 불고기 재료끼리 → **결과물(아웃풋) 생산에 유리**
- PARA는 밀키트 방식과 유사 — 노트를 활용해 아웃풋을 만들어 내도록 설계

## PARA의 네 가지 분류 (실행도 높은 순)

### 1. Project (프로젝트)
- **끝이 있다** — 구체적인 달성 목표와 종료 날짜 존재
- 현재 전념하고 있는 가장 중요한 일들
- 예시: 자격증 시험 공부, 블로그 글 작성, 제품 런칭, 전자책 출간, 이사하기

**자주 하는 실수:**
- 이사하기 프로젝트 진행 중, '부동산 계약 시 주의사항' 노트를 주제 폴더(부동산)에 넣는 것
- PARA에서는 **현재 진행 중인 프로젝트 폴더가 우선** → 이사하기 프로젝트 폴더에 넣어야 함

**프로젝트 완료 후 처리:**
- 완료된 프로젝트 안의 노트들을 하나씩 검토
- 관련 다른 프로젝트, 영역, 자원 폴더로 이동
- 어디에도 속하지 않으면 **아카이브**로
- 즉, 노트는 실행도에 따라 **살아 움직이는** 시스템

### 2. Area (영역)
- 내가 진행 중인 **역할 또는 책임**에 해당
- **끝이 없다** — 프로젝트와의 가장 명확한 구분점
- 예시: 직업, 건강, 경제 계획, 가족
- 대부분의 프로젝트는 어느 영역을 발전시키기 위해 생겨남

### 3. Resource (자원)
- 프로젝트·영역에 해당하지 않지만 **관심 있는 주제**들
- 영역과의 차이: **책임의 유무**
  - 자원 항목은 못하거나 안 해도 별 문제 없음
- 개인마다 분류가 다름:
  - 작가에게 '글쓰기' = 영역 / 일반인에게 = 자원
  - 취미 블로그 = 자원 / 부업 블로그 = 영역

**자원에서 흔한 오해 (한국 사용자):**
- '자원'이라는 이름 때문에 참고자료·영상자료·논문 등을 모으는 곳으로 착각
- 실제로는 참고자료도 **실행도 기준**으로 분류 — 현재 프로젝트에 유용하면 프로젝트 폴더로

### 4. Archive (아카이브)
- 위 세 가지 어디에도 속하지 않는 것들
- **실행도 = 0**
- 관심도 낮지만 혹시 몰라 보관하는 항목들
- 완료된 프로젝트, 중단된 프로젝트도 포함
- **버려진 창고가 아님** — 과거 작업물을 레버리지하는 원동력
  - 예: 이전 성공한 보고서·기획서 → 새 프로젝트에 형식·내용 활용
  - 예: 지난 여행 준비물 목록 → 새 여행에 재활용

## PARA의 강점

- 시간이 지날수록 노트 모음이 **나 자체를 담게 됨**
- PARA 순서대로 정리된 대시보드를 보면:
  - 현재 집중하는 중요한 일 (프로젝트)
  - 나의 책임 영역
  - 나의 관심사 (자원)
  - 과거 기록 (아카이브)
  - → 모두 한눈에 파악 가능
- **세컨드 브레인(Second Brain)** — 외부에 위치한 나의 두 번째 뇌

## 노션 PARA 템플릿 사용법

저자가 직접 제작한 노션 템플릿(캡션에서 무료 다운로드) 시연

### 기본 구조
- **인박스(수집함)**: 정리되지 않은 새 노트가 모이는 곳
- 노트가 프로젝트·영역·자원으로 배정되면 수집함에서 사라짐
- 주기적으로 수집함을 비워주는 것이 핵심

### 노션의 장점 vs. 일반 폴더 방식
- 일반 폴더(윈도우·옵시디언): 폴더를 직접 찾아 노트를 복사·붙여넣기 필요
- 노션: 폴더 없이 **데이터베이스 + 검색·선택** 방식 → 훨씬 편리
- **하나의 노트가 여러 폴더에 동시 소속 가능** (프로젝트 + 영역 동시 연결)
  - 예: '후코 겨울 여행' 프로젝트와 '여행' 자원 모두에서 같은 노트 접근 가능
- **노트 위치 이력 보존**: 폴더 이동 시 위치 정보 유실 문제 해결 — 이 노트를 예전에 어떤 프로젝트에서 썼는지 확인 가능

### 노트 페이지 기능 (4가지)
1. **영역·자원 선택** — 선택 시 수집함에서 사라지고 해당 페이지에 표시, 복수 선택 가능
2. **고정하기** — 홈화면 피드에 즐겨찾는 노트 고정
3. **연결된 프로젝트** — 연결 개수 표시, 프로젝트 추가·해제
4. **연결된 노트** — 관련 노트끼리 연결해 지식 확장

### 프로젝트 페이지 기능 (5가지)
1. **상태 관리**: 진행 중 / 시작 안 함 / 완료 / 중단 → 진행 중이 아니면 자동으로 아카이브로
2. **영역·자원 연결**: 해당 영역 페이지에서 관련 프로젝트 한눈에 파악
3. **시작일·종료일**: 홈화면에서 **D-day** 표시, 종료일 미설정 시 알림
4. **완료하기**: 상태를 완료로 변경 + **회고 노트** 자동 생성 (필수는 아니지만 권장)
5. **연결된 노트 목록**: 하단에 표시, 새 노트 추가 시 프로젝트 자동 연결

### 영역·자원·아카이브
- 동일한 데이터베이스에서 **상태 변경**으로 전환
- 영역 페이지에서 프로젝트를 **드래그 앤 드롭**으로 상태 변경 가능
- 자원은 영역과 동일한 구조 — 차이는 책임 유무뿐

### 아카이브 항목
- 시작 안 한 프로젝트
- 중단된 프로젝트
- 완료된 프로젝트
- 아카이브 처리된 영역·자원

### 템플릿 메뉴
- **시작하기 페이지**: 초기 세팅 가이드
- **데이터베이스 페이지**: 필터 없는 전체 원본 데이터 조회

## 세컨드 브레인 OS 템플릿과의 관계

- 이 PARA 노트 템플릿은 세컨드 브레인의 **미니 버전**
- 세컨드 브레인 OS = PARA + **GTD 할일 관리 시스템** 추가 → 인생 전체를 하나의 노션 템플릿에서 관리
- 기존 세컨드 브레인 OS 데이터를 PARA 노트 템플릿으로 이전 가능

**데이터 이전 방법:**
1. 데이터 목록 탭 열기 → 프로젝트 탭 클릭
2. 전체 선택 체크박스로 모든 데이터 선택
3. 합계 개수 = 선택된 개수 확인 (불일치 시 '더 불러오기'로 전체 로드)
4. 점점점(···) → 옮기기 → 대상 템플릿의 프로젝트 데이터베이스 선택 (경로 확인 필수)
5. 영역·자원·노트도 동일하게 반복
