## 문제 공감 — CLAUDE.md만으로는 부족한 이유

- **클로드 코드** 등 AI 에이전트 도구를 쓸 때 보통 `CLAUDE.md`(또는 `agent.md`, `gemini.md`)로 세팅하지만, 대화가 끝나면 지식이 사라짐
- 재활용 가능한 나의 정보·지식을 축적하기 어렵고, AI가 나의 관점과 가치관을 지속적으로 파악하기 힘듦
- **세컨드 브레인**이 필요하지만 체계를 만들다 포기하는 경우가 많음

## 해결 조합 — 왜 옵시디언인가

- **옵시디언**: 순정 마크다운 기반 지식 관리 앱 → AI 도구가 가장 잘 소화하는 포맷
- 안드레 카파시(OpenAI 공동 창립자)가 옵시디언을 "AI를 위한 **LLM 프런트엔드**"라고 언급(2026-04-02경) → 폭발적 관심
- 세계 관찰 → 옵시디언 마크다운 입력 → 클로드 코드 프로세싱 → 구조화 문서(MD) → DOCX·PPT·PDF·MP4 등 다양한 산출물 생성 흐름

## Gold In, Gold Out — 목적 있는 수집

- **목적 있는 수집**: 내가 왜 수집했는지 설명할 수 없으면 진짜 의도적 수집이 아님
- 목적 없이 수집하면 쓰레기 데이터 → 산출물 품질 저하 (Garbage In, Garbage Out)
- 저자는 이를 **"Gold In, Gold Out"**으로 3년 전부터 명명
- 목적 있는 수집이 쌓여야 AI가 활용할 수 있는 골드 데이터가 늘어남

## LLM Wiki란 무엇인가

- 안드레 카파시가 2026-04-03 트윗(스윗)으로 제안한 지식 관리 프레임워크
- 바이브 코딩(2025-02), 컨텍스트 엔지니어링(2025-06)에 이어 세 번째 메타 트렌드로 부상
- **핵심 흐름**:
  1. 인터넷·논문·영상·이미지·아티클 등에서 목적 있는 수집 → `raw` 폴더 저장
  2. **LLM 에이전트**(클로드 코드 등)가 소스를 읽고 기존 지식과 연관성 확인 → 링크 생성 → 개념·엔티티 분리
  3. `wiki` 폴더에 목차·인덱스·통합 문서 형태로 정리 → 루프 반복
  4. 새 자료가 들어올 때마다 기존 위키와 연결·업데이트
- **스키마**: LLM이 매번 랜덤으로 처리하지 않도록 나의 목적·기준·처리 방법을 고정한 규칙
- 완전히 구조화된 것이 아니라 큰 아이디어 제시 → 각자 자신만의 버전으로 발전

## LLM Wiki vs RAG — 근본적 차이

| 항목 | RAG | LLM Wiki |
|------|-----|----------|
| 셋업 복잡도 | 높음 (임베딩 모델, 벡터 DB 필요) | 낮음 (마크다운 파일만) |
| 인프라 | 벡터 DB 필수 | 로컬 폴더면 충분 |
| 검색 신뢰도 | 문서 품질·임베딩에 의존, 누락 발생 가능 | 목차·스키마 기반으로 상대적으로 높음 |
| 지식 축적 방식 | 문서 추가 시 재임베딩 필요 | 인제스트 시 위키에 연결·누적 |
| 복리 효과 | 상대적으로 약함 | **복리형** — 연결이 쌓일수록 가치 증가 |

## 세팅 실연 — 단계별 과정

### 1. 옵시디언 볼트 생성
- 새 볼트 생성(예: `llm-wiki-brain-trinity`) → 빈 마크다운 환경 준비
- 클로드 코드에서 해당 폴더를 열어 세팅 진행

### 2. 핵심 맥락 작성 (나를 파악하는 인터뷰)
- 새 노트에 세 가지 핵심 질문에 STT로 답변:
  - **나는 누구인가** (역할·강점·핵심 가치관)
  - **왜 기록하고 싶은가** (동기·비전)
  - **어떤 아웃풋을 만들고 싶은가** (유튜브 롱폼·인스타 숏폼·스레드·링크드인 등)
- 예시 저자 맥락: 브레인 트리니티 브랜드 운영, 개인 지식 관리·AI 본질 전달, 롱폼 영상·카드뉴스·숏폼 글 제작
- 프롬프트: `"나의 핵심 맥락을 읽고 나를 더 깊이 이해하기 위해 인터뷰해줘"`
- 인터뷰 결과로 핵심 맥락 노트 자동 업데이트

### 3. CLAUDE.md 자동 생성
- 프롬프트: `"핵심 맥락 파일을 바탕으로 CLAUDE.md를 만들어줘 (작업 규칙, 나는 누구인가, 기대 사항 등 포함)"`
- 생성된 CLAUDE.md: 나의 목적·역할·AI에게 기대하는 것·작업 규칙 포함
- 필요 시 지속적으로 업데이트

### 4. 폴더 구조 & 위키 스키마 세팅
- LLM Wiki 원문(카파시 트윗 전문)을 프롬프트에 붙여넣어 나의 목적에 맞게 폴더 구조 생성 요청
- 생성된 폴더: `raw`(수집 소스), `wiki`(정리된 지식), `output`(산출물), 각 폴더별 `CLAUDE.md`
- 각 폴더의 `CLAUDE.md`로 해당 폴더 맥락을 AI가 추가 파악 가능 → 맥락 풍부화

### 5. 옵시디언 웹 클리퍼 템플릿 커스텀
- 옵시디언 웹 클리퍼(크롬 익스텐션)로 인터넷 자료를 마크다운으로 수집
- 기본 템플릿을 JSON으로 내보내기 → 클로드 코드에 제공 → LLM Wiki 스키마에 맞는 커스텀 템플릿 생성 요청
- 생성된 템플릿 종류: **아티클·유튜브·팟캐스트·책·연구자료** 5종
- 각 템플릿을 웹 클리퍼 옵션에서 임포트 → 사이트별 자동 트리거 설정
- 유튜브 영상 클리핑 시 **트랜스크립트까지 자동 수집**

### 6. 인제스트 — 목적성 있는 지식 소화
- 프롬프트 예시: `"raw 폴더에 저장한 파일들을 읽고 나에게 질문한 후 위키에 반영해줘"`
- **핵심**: 인제스트 전 내가 왜 이 자료를 수집했는지 코멘트 입력 필수
  - 예: "LLM Wiki를 공부하고 브레인 트리니티 프로그램과 연결하고 싶어서 수집"
- AI가 소스를 읽고 → 엔티티(LLM, RAG, Codex 등) 분리 → 기존 위키와 연결 → wiki 폴더에 개념 문서 생성
- 대량 수집 시 클로드 프로 플랜 토큰 소모 주의

### 7. 스킬 생성 & 워크플로우 자동화
- 인제스트 과정을 2회 이상 수행 후 AI가 패턴 학습 → **`/ingest` 스킬** 생성 요청
- 추가 생성 스킬:
  - **`/query` (코리 스킬)**: 위키 내용을 기반으로 질문에 답변 (벡터 DB 없이 마크다운 기반 검색)
  - **`/lint` (린트 스킬)**: 위키 전체 점검·최신화·깨진 링크 수정 (도서관 정리 스킬)
- 스킬은 대화 없이 슬래시 커맨드 하나로 자동 실행

### 8. 위키 쿼리 & 신테시스
- `/query "브레인 트리니티와 LLM 위키의 상관성 알려줘"` → 위키 내 관련 문서 검색 후 답변
- 쿼리 결과를 `synthesis` 폴더에 저장 가능 → 여러 소스를 통합한 보고서 형태
- 신테시스는 참고 자료이며 직접 활용이 아닌 새로운 지식 창출의 재료로 사용

## Graphify — 지식 그래프 추가

### LLM Wiki의 한계
- 문서가 많아질수록 키워드 기반 텍스트 검색의 효율 저하
- 옵시디언의 그래프 뷰는 시각화 용도이며, AI가 그래프 정보를 직접 활용하지 않음

### Graphify 개념
- 카파시 LLM Wiki 트윗 후 48시간 내에 누군가 개발한 오픈소스 도구
- **마크다운 위키를 그래프 DB로 변환** → AI 코딩 어시스턴트가 노드·엣지 기반으로 활용
- 공식 설명: "Knowledge graphs for AI coding assistants"

### 설치 & 사용
- 사전 조건: Python 설치
- 설치: `pip install graphify`
- 사용: `graphify wiki` (위키 폴더 지정) → 그래프 생성
- 출력물:
  - `graph.json`: AI가 활용하는 그래프 데이터
  - `graph.html`: 사람이 보는 시각화 (노드·엣지·커뮤니티 클러스터 표시)
  - 마크다운 리포트: 옵시디언 내에서 확인 가능
- 업데이트: `graphify wiki --update` → 변경된 부분만 재추출

### Graphify 쿼리
- `graphify query "브레인 트리니티와 LLM의 근본적 차이"` → 그래프 탐색 결과 기반 답변
- 텍스트 기반 검색 대비 노드·엣지 관계를 활용한 더 정교한 연결 파악
- 예시 답변: "같은 문제를 다른 레이어에서 푸는 관계, 추상화 수준과 관심사 방향이 다름"

## 핵심 원칙 & 마무리

> "LLM이 알아서 해 주는 게 아닙니다. 제일 중요한 건 나의 목적입니다. 목적 없이 왜 수집했는지 답할 수 없으면 LLM Wiki를 제대로 활용하는 게 아닙니다."

- 이 조합(옵시디언 + 클로드 코드 + LLM Wiki + Graphify)이 **현재 메타**로 부상 중
- 지식의 복리 효과: 수집·연결이 쌓일수록 AI가 더 정밀하게 활용 가능
- 프롬프트 모음은 영상 설명란 링크에서 공유
